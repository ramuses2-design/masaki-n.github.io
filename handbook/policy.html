2. 使命・ビジョン・価値観（核メッセージ）

使命：製品を検査するだけでなく、「お客様の事業に貢献する品質」を実現する。 検査・試験にとどまらず、不良の原因分析・工程改善・取引先連携まで、ものづくり全体に関わる。

目指す状態：検査で不良を「見つける」より、そもそも不良を出さない仕組みをつくる。

根底の価値観

  • 顧客視点 — 基準を満たすかだけでなく、その製品を使うお客様の立場で考える。
  • 現場主義 — 答えは現場にある。データと事実をもとに、現場で確かめる。

3. 部下に一貫して伝える「ブレ防止 6原則」

1on1・指導で繰り返し使う共通言語。ここだけは全員に同じ言葉で。

  1. 顧客視点 … 「この判断、お客様の事業から見てどうか？」を最初に問う。
  2. 現場主義 … 机上で決めず、現物・現場・データで確かめてから語る。
  3. クレームは出さない … 不良を出さない仕組みで、お客様に迷惑をかけない。検査で止めるのは最後の砦、品質は前工程で作り込む。
  4. データ・事実で語る … 感覚ではなく数字で。「なんとなく」を許さない、責めない。
  5. 改善に意図的に時間を使う … 日々の検査に追われて改善ゼロにしない。改善は“業務”であって余技ではない。
  6. 横展開を意識する … 一件の改善・是正をその場で終わらせず、他の品種・工程・担当へ展開する。

5. ルール・方針（ここのブレを無くす）

5.1 品質の合否・出荷可否の判断と権限

  • 通常合否は担当者が判定してよい。 データ（規格・検査基準）に基づいていれば、GLへは報告のみでOK。
  • 製品仕様書外であっても、お客様の用途によっては特採（特別採用）申請を行う。仕様外＝即NGではない。
  • 根拠がなければ交渉しない。 データ・事実の裏づけがない要求や判断は、お客様にも社内にも持ち込まない。
  • GL（私）が最終判断する観点：①お客様との信頼関係、②売上への影響、③費用対効果。この3点が絡む判断は必ずGLへ。
  • 判断は必ず規格・データ・記録に紐づける。「前もこうだったから」で決めない。

5.2 エスカレーション（報連相）ルール　【要確認】

“即・私へ”上げる事象（時間を置かない／悪い情報ほど早く）：

  • 顧客クレーム・市場不良・不良流出の疑い
  • 監査・審査での指摘、法規制（RoHS/REACH 等）に関わる懸念
  • ライン停止・出荷遅延につながる重大不良
  • 安全・コンプライアンスに関わる事象
上記以外は週次報告でよい。「迷ったら早めに一報」を基本姿勢にする。 連絡手段：メール／Teams／電話／口頭（緊急度に応じて）。一次受けはGL（私）に直接。間に挟まず最短で上げる。

5.3 会議体・レポートのリズム

  • 週次（グループ内）：参加＝グループ員のみ。進捗・不良傾向・負荷状況・困りごとを共有。
  • 月次（部門横断）：参加＝製造係長＋他グループリーダー＋事業部長。KPI・重要案件・改善ロードマップの進捗を報告。

5.4 記録・文書ルール　【要確認】

  • 検査表・測定データ・判定根拠は証跡として必ず残す（後で第三者が追える状態に）。

要確認正式な記録様式・保管期間・アクセス権限。

6. 私（masaki）が裏で動いていること

部下から見えにくい“マネジメントの実務”を明示し、「なぜその指示か」の誤解を減らす。

6.1 対外・上位調整（私が引き受けていること）

  • 緊急対応・交渉が必要な顧客窓口：トラブル・クレーム・難しい交渉はGLが前に出る。
  • 監査対応：審査・顧客監査の窓口。
  • 上層部報告：事業部長への報告（月次ほか）。要求基準を現場の言葉に翻訳して下ろす。
  • スタンス：「現場が検査・改善に集中できるよう、対外の調整は私が引き受ける」。

6.2 人員・負荷マネジメント

  • 工数の偏りの是正：担当者間で年間工数に最大約3.4倍の差。月間キャパ×稼働率で判断し、移管可能な業務を過負荷者から余力者へ再配分。
  • 属人化の解消：重要業務に副担当を必ず設定、スキルマップ化し「最低2名でカバー」を目指す。
  • 目的は管理ではなく、疲弊・離職リスクの低減と事業継続性。

6.3 中長期の仕掛け（改善ロードマップ）

  • 業務管理アプリへの移行：Excel台帳 → 割当・工数・集計・権限・通知を備えたアプリへ（暫定改善 → 本構築 → データ移行 → 本稼働・継続改善）。
  • 検査表作成の標準化・テンプレ化・一部自動化（全工数の約44%を占め偏りが大きいため）。
  • 改善活動の工数確保：各人の工数の5〜10%を改善に意図的に配分し、ダッシュボードで可視化・目標管理。
  • 計画対実績の導入：計画だけでなく実績・乖離を見える化し、見積り精度を上げる。
要確認優先順位・時期・追加の仕掛け。

6.4 育成・評価の方針

  • 仕組みとして 1on1・スキルマップ運用・評価軸 を回す。
  • 副担当・スキルマップでバックアップ体制と成長機会を同時に作る。
  • 新任は在籍状況を明示し育成計画に載せる（戦力把握とセット）。
要確認1on1の頻度、スキルマップの具体項目、評価軸の中身（確定したらここを埋める）。

8. 用語メモ（社内の言葉を揃える）

  • QA（品質保証） … 出荷後も含めてお客様に品質を“約束”する広い活動。
  • QC（品質管理） … 主に工程内で基準を満たすかを管理する活動。
  • 属人化 … 特定業務が特定個人に依存している状態。副担当で解消する。
  • 特採（特別採用） … 規格外でも条件付きで採用すること。出荷可否は要・上位判断。
  • 目付（めつけ／坪量） … 単位面積あたりの質量（g/m²）。不織布の基本特性で、多くの不良と相関する。
  • 地合い（じあい） … 繊維分布の均一さ。ムラ・透け・強度ばらつきの元になる。
  • 通気度 … 空気の通りやすさ。フィルター・衛生材など用途で重要な機能特性。
  • MD／CD … 流れ方向（Machine Direction）と幅方向（Cross Direction）。強度・伸度は方向で差が出る。
  • 4M … Man（人）・Machine（機械）・Material（材料）・Method（方法）。変化点管理の基本軸。
  • 8D／CAPA … 是正処置の型。8D＝8ステップ、CAPA＝是正・予防処置。B4で使う。
  • MSA … 測定システム解析。測定のばらつき（Gage R&R 等）を評価する手法。
  • chemSHERPA … 製品含有化学物質の情報を伝達する共通スキーム。IMDSと並ぶ代表的な仕組み。
  • SPC（統計的工程管理） … 管理図などでばらつきを監視し、異常を早期に検知する手法。
  • Cp／Cpk（工程能力指数） … 規格に対する工程のばらつき余裕度。Cpkは中心ずれも考慮する。
  • ポカヨケ … 間違いを物理的・仕組み的に防ぐ工夫。取り違え・付け忘れの防止に使う。
  • スパンボンド／メルトブロー … 代表的な不織布の製法。長繊維系と極細繊維系で特性が異なる。
  • なぜなぜ分析（5 Whys） … 「なぜ」を繰り返し真因に迫る手法。B3/B4の原因究明で使う。
  • 限度見本 … 合否の境目を示す実物見本。目視検査の基準を揃えるために用いる。
  • ΔE（色差） … 2色の差を数値化した指標。色調・白度の合否管理（TS18）で使う。
  • PFMEA（工程FMEA） … 工程の故障モードと影響を事前に洗い出し、優先度をつけて対策する手法（T11）。
  • 層別監査（LPA） … 役職を横断した層別の定期工程監査。標準・現物・記録を現場で確認する（D8）。
  • FIFO（先入れ先出し） … 先に入れた在庫から先に使う運用。滞留・期限切れの防止に有効（P9）。
  • コントロールプラン … 工程ごとの管理項目・基準・方法・頻度をまとめた管理計画表。
  • 圧力損失（圧損） … 通気時の圧力の落ち込み。通気度・ろ過性能の指標（TS19）。
  • 剛軟度（剛軟性） … 布・不織布の曲げやすさ（硬さ・柔らかさ）の指標。風合い・ドレープ性の評価に使う（TS21）。
  • スリット／断裁 … 広幅の原反を製品幅・長さに切り分ける工程。切断面の毛羽・バリ・寸法品質に影響（TS20）。
  • SPC管理図（Xbar-R 等） … 工程のばらつきを時系列で監視し、異常を早期検知する管理図（T12）。
  • リサイクル素材／GRS … 再生原料の使用。GRS等の認証は再生材含有率とトレーサビリティを求める（S12）。
  • 選別（ソーティング） … 良品・不良品を基準で仕分ける作業。流出後の切り分け・回収判断で行う（B9）。

---

  • エンボス／型付け … ロールの凹凸で不織布に模様・立体を付与する加工。欠け・つぶれ・位置ズレが外観/風合いに影響（TS22）。
  • 紙管／コア … 巻取りの芯となる管。強度不足・変形はコアマークや巻き崩れの原因（TS23）。
  • トレーサビリティ … 原料〜工程〜出荷をロット・識別で追える状態。流出時の範囲特定に不可欠（T13・B9）。
  • チョコ停 … 短時間の設備停止。頻発すると品質異常・生産性低下の兆候になる（M13）。
  • 通商措置／関税 … 輸入品にかかる追加関税等の政策。調達コスト・供給に影響し、調達リスク管理の対象（P11）。
  • ラミネート／貼合 … フィルム等と貼り合わせて複合材にする加工。層間剥離・気泡・シワが品質に影響（TS24）。
  • 検針／金属検出 … 折れ針・金属異物を検出する工程。テストピース照合で感度を担保する（TS25）。
  • SCAR（是正処置報告書） … 顧客へ提出する不適合の是正報告様式。8Dと同様の型で作成する（B8）。
  • 受入不良率（PPM） … 受入100万個あたりの不良数。サプライヤー品質のスコアリングに使う（P12）。
  • 相見積り（あいみつ） … 複数社から見積りを取り比較すること。価格以外の品質価値の訴求が重要（S14）。
