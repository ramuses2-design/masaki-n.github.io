# 品質保証グループ 読者別ハンドブック（連携ガイド）

> **位置づけ**：品質保証グループ（QA）から、**営業・技術・製造・購買**の各部門、および**外注先（お取引先）**へ向けた連携の手引き。
> 「品証と、いつ・何を・どう連携するか」を場面別にまとめたもの。方針マスターの**ブレ防止6原則**と**エスカレーション基準**が土台。
> **考え方の核**：品質は検査で「見つける」より、**前工程で作り込む**。だから各部門・取引先との連携が品質の大半を決める。
> **凡例**：`★`＝特に外すと事故につながる重点項目。`【要確認】`＝GL（masaki）の確定が必要な箇所。各トピック末尾の「関連場面」から、品証内プレイブック（A〜D）へ辿れる。

## 読者別マップ
- **🤝 営業向け**：見積前確認／要求仕様の連携／クレーム一次対応／特採／出荷可否／監査対応／提出書類／仕様変更対応
- **🛠️ 技術向け**：新製品立ち上げ／4M変更／検査基準／設備導入／工程能力・データ活用／化学物質の設計織り込み／再発防止／治具・型管理
- **🏭 製造向け**：日常検査／異常時初動／変化点申告／測定器・校正／不良報告／5S・監査準備／切替初物／作業標準・力量
- **🧺 購買向け**：サプライヤー選定／受入検査／4M変更管理／化学物質エビデンス／不適合対応／監査・改善
- **📦 外注先向け**：求める品質の基本／提出書類／変更の事前連絡／不適合時の連絡／化学物質対応／納入・識別

# S. 営業向け

> **合言葉：「品質は“約束”。約束できないことは持ち帰る」**
> 営業は、お客様の声が最初に入る窓口。ここで拾う情報と、ここで交わす約束が、品質トラブルの起点にも、予防の起点にもなる。困ったら一人で抱えず、早めに品証（GL）へ一報を。

## S1. 引合い・見積前の品質確認
**目的**：受注前に「作れる・測れる・約束できる」を品証と確認し、無理な約束を防ぐ。

**確認して品証へ渡すこと**
- [ ] 要求仕様・規格・図面・適用法規を入手し品証へ共有 ★
- [ ] 用途・使用環境・お客様の重要特性（どこが効く品質か）を聞き取る
- [ ] 当社の実力（工程能力・検査可否・測定可否）を品証に確認 ★
- [ ] 特殊要求（清浄度・化学物質の非含有・トレーサビリティ・提出書類）の有無を確認
- [ ] 「品質面で約束できる範囲」を品証と握ってから見積る

**やってはいけない（NG）**
- 実力を確認せずに納期・スペックを口頭で約束する／「たぶん大丈夫」で受ける

関連場面: C2, C4

## S2. 顧客要求・仕様書の受け取りと連携
**目的**：お客様の要求を、抜け・取り違えなく品証が使える形で渡す。

- [ ] 仕様書・要求規格・検査要求・提出書類（成績書・非含有保証等）を一式で受領 ★
- [ ] 版数（改訂）を必ず最新で共有し、旧版と取り違えない ★
- [ ] 図面・規格の「特別特性」「重要保安部品」等の指定を見落とさない
- [ ] 受領した要求を品証へ連携し、対応可否・条件の回答を得る
- [ ] お客様固有の帳票・Webポータル・提出期限があれば品証と共有

関連場面: C4, D4, D5

## S3. クレーム・苦情の一次対応
**考え方**：「まず事実、約束は持ち帰る」。対外の最終窓口はGL／品証と連携して動く。

- [ ] 事実を聞き取る（現品・ロット・数量・発生状況・お客様の困りごと）★
- [ ] その場で原因・可否・賠償を約束しない ★
- [ ] 即、品証GLへ連携（クレーム・市場不良は**即エスカレーション**事象）★
- [ ] 暫定対応（代替品手配・選別）の要否を品証と決める
- [ ] 回答期限をお客様と握り、社内へ共有する

**やってはいけない（NG）**
- 憶測で原因を語る／確定していないことを約束する／独断で「無償対応します」と言う

関連場面: B2, B6, B1

## S4. 特採（特別採用）・仕様外品の取り扱い
**考え方**：規格外＝即NGではない。ただし**お客様承認なしに出荷可否を変えない**。

- [ ] 仕様外の可能性は品証へ相談（営業単独で判断しない）★
- [ ] 特採は必ずお客様の承認を取得する（無断で採用しない）★
- [ ] 安全・法規に関わるものは特採の対象外（絶対）★
- [ ] 適用範囲・数量・期限を限定し、恒久化させない
- [ ] 申請・承認・適用の記録を残す

関連場面: B5

## S5. 納期・出荷可否の約束の線引き
**目的**：品質判断（出荷可否）と営業判断（納期）の境界を守る。

- [ ] 出荷可否は、品質データに基づく品証の判断を待つ ★
- [ ] 品質保留中のロットを「間に合わせ」で出さない ★
- [ ] 短納期の依頼ほど早く品証へ共有（無理な短納期は品質リスク）
- [ ] GLが最終判断する観点（①信頼関係 ②売上影響 ③費用対効果）に関わる案件は必ずGLへ

関連場面: B1, A3

## S6. 監査・品質会議・顧客訪問での連携
**目的**：お客様の品質要求・評価を、社内へ正しく持ち帰る。

- [ ] 顧客監査・工場訪問の予定は早めに品証へ共有（準備に時間が要る）★
- [ ] お客様の品質評価・要求の変化・競合情報を品証へフィードバック
- [ ] 品質会議の決定事項・宿題を記録し、期限を管理する

関連場面: D3

## S7. 提出書類・品質エビデンスの取り交わし
**目的**：お客様へ出す品質エビデンスを、正確・期限どおりに用意する。

- [ ] 必要書類（検査成績書・ミルシート・非含有保証・PPAP／初回流動）を品証と確認 ★
- [ ] 提出期限・提出先（帳票／Webポータル）を管理する
- [ ] 記載内容の最終確認（ロット・規格・測定値の整合）★
- [ ] 発行・提出の記録を残す（後で追える状態に）

関連場面: D5, S2

## S8. 顧客要求・仕様の変更対応
**考え方**：変更は「受けたら終わり」でなく、社内へ確実に展開して初めて完了。

- [ ] 変更（規格・図面・検査要求）を最新版で受領し版数を管理 ★
- [ ] 変更内容を品証・技術・製造へ速やかに展開 ★
- [ ] 対応可否・切替時期・在庫（旧仕様品）の扱いを社内と確認
- [ ] お客様へ切替時期・条件を回答する

関連場面: C1, C4, S2

# T. 技術向け

> **合言葉：「前提が変わる＝新しいリスクが生まれる」**
> 技術（生産技術・開発）の設計・工程・設備の決定は、品質の作り込みそのもの。品証は「検査で止める前に、工程で作り込む」を掲げている。変更・立ち上げ・基準づくりを、品証と“先に”握るほど後戻りが減る。

## T1. 新製品・量産立ち上げ（APQP／PPAP 的）
**目的**：立ち上げ段階で品質を作り込み、量産後のクレームを防ぐ。

- [ ] 顧客要求・品質特性・規格を初期に明確化し品証と共有 ★
- [ ] 設計審査（DR）・FMEA的なリスク抽出に品証を参画させる ★
- [ ] QC工程表・検査基準・限度見本を品証と作成
- [ ] 初物確認・工程能力（Cp/Cpk）を確認してから量産移行 ★
- [ ] 顧客承認（PPAP／初回流動）の要否を確認
- [ ] 初期流動管理（立ち上げ直後の重点監視）を設定 ★

関連場面: C2, C4, D6

## T2. 4M変更管理（人・機械・材料・方法）
**考え方**：最大のリスクは“暗黙の変更”。**無断変更は禁止**。

- [ ] 人・機械・材料・方法・測定・環境の変更は、事前に品証へ ★
- [ ] 変更前に影響評価・リスクアセスメントを行う
- [ ] 顧客承認・4M変更届の要否を確認 ★
- [ ] 妥当性確認（初物・工程能力・試験）後に切り替える
- [ ] 変更履歴を記録する（いつ・何を・なぜ・誰が）

**やってはいけない（NG）**
- 「良くするだけだから」と、条件・材料・治具を無断で変える ★

関連場面: C1

## T3. 工程・検査基準の制定／改訂
**目的**：曖昧さのない、根拠のある基準を品証と合意する。

- [ ] 基準の根拠（規格・顧客要求・不具合履歴）を明確にする
- [ ] 測定方法・判定基準を曖昧さなく定義 ★
- [ ] 限度見本を整備（官能・外観検査は特に）
- [ ] 版管理（最新版のみ現場へ、旧版は回収）★
- [ ] 改訂時は影響評価と顧客承認の要否を確認

関連場面: C4, D1, D5

## T4. 設備の導入・改善
**目的**：「安全に作れる」「品質を測れる・維持できる」を設計段階から織り込む。

- [ ] 目的・解決する品質課題を明文化 ★
- [ ] リスクアセスメント・安全対策（非常停止・ロックアウト・安全カバー）★
- [ ] 環境影響（化学物質・排気排水・廃棄物：ISO 14001）を確認
- [ ] 据付／稼働／性能確認（IQ／OQ／PQ 的）★
- [ ] 測定設備は校正・MSA（Ga R&R）で「測れる」を証明 ★
- [ ] 作業手順書・教育を整備（副担当を含め属人化させない）

関連場面: C3-a, C3-b, D2

## T5. 工程能力・初期流動・データ活用
**目的**：数字で品質を語り、不良の予兆を工程へ返す。

- [ ] 工程能力（Cp／Cpk）で実力を把握 ★
- [ ] ばらつき・測定の再現性（MSA）を考慮する
- [ ] 検査データを蓄積・分析し、不良の予兆を工程へフィードバック（品証の三本柱＝AI・画像検査／データ活用）
- [ ] 初期流動で異常を早期に検知する

関連場面: D6, C2

## T6. 化学物質・環境法規の設計織り込み
**目的**：後戻りを防ぐため、材料選定の段階で法規と顧客要求を満たす。

- [ ] 適用法規（RoHS・REACH・各国規制）を材料選定時に確認 ★
- [ ] 成分情報（SDS・IMDS・chemSHERPA）を入手・管理
- [ ] 含有調査・非含有保証の要否を品証と確認 ★
- [ ] 材料・工程の変更時は再調査（変更管理と連動）

関連場面: D4, C1

## T7. 不具合の再発防止と設計・標準へのフィードバック
**目的**：起きた不具合を、次の設計・工程標準に取り込み、再発を断つ。

- [ ] 是正で判明した弱点をFMEA・設計標準へ反映 ★
- [ ] QC工程表・作業標準・限度見本を更新する
- [ ] 類似製品・類似工程へ水平展開 ★
- [ ] 反映結果を記録し、次の立ち上げで参照できるようにする

関連場面: B4, C2, C4

## T8. 治具・型・設備の維持管理
**目的**：「作る道具」の劣化・変化を管理し、品質のばらつきを防ぐ。

- [ ] 治具・型の識別・履歴管理（使用回数・メンテ）★
- [ ] 定期点検・メンテ基準を設定する
- [ ] 摩耗・劣化による品質影響を監視（初物・工程能力）
- [ ] 改造・修理は4M変更管理にのせる ★

関連場面: C3-b, C1, D2

# M. 製造向け

> **合言葉：「異常はまず止める、それから直す」／「自己流にしない」**
> 品質は検査で見つける前に、工程で作り込む。現場の一つひとつの操作・記録・気づきが品質そのもの。難しい理屈より「これだけは守る」を優先。**悪い情報ほど早く**、ためらわず上げてほしい（責めるためではなく、仕組みで直すため）。

## M1. 日常の検査・検査表運用
**目的**：決めたとおりに測り、そのまま記録する。

- [ ] 始業前点検（測定器・標準・限度見本）★
- [ ] 検査基準どおりに実施する（自己流にしない）★
- [ ] 抜取／全数の区分を守る
- [ ] 記録は即時・正確に（後でまとめ書きしない）★
- [ ] ロット・日時・担当のトレーサビリティを残す

**やってはいけない（NG）**
- 「いつもと同じだから」で測定を省く／記録を後でまとめて埋める

関連場面: D1

## M2. 異常が出たときの初動（まず止める）
**合言葉**：まず止める、それから直す。一人で抱えない。

- [ ] 現品・現物を動かす前に記録／写真を撮る ★
- [ ] 止める・隔離する（赤札・識別で区別）★
- [ ] すぐ連絡する（上長・品証へ）★
- [ ] 勝手に手直し・選別して流さない ★
- [ ] いつから・どれだけ（数量・期間）を分かる範囲で伝える

関連場面: B2, B1

## M3. 4M変化点の申告（無断で変えない）
**考え方**：いつもと違うことは、良いことでも必ず申告。

- [ ] 材料ロット替え・設備調整・人の交代・方法変更は事前に申告 ★
- [ ] 「良くするつもりの変更」も無断でやらない ★
- [ ] 変化点があった日は記録に残す
- [ ] 異常の芽（音・におい・ばらつき・手ごたえの違い）に気づいたら共有する

関連場面: C1, B3

## M4. 測定器・校正の順守
**目的**：「正しく測れる道具」でしか判定しない。

- [ ] 校正期限切れの測定器を使わない ★
- [ ] 使用前点検（ゼロ点・ばらつき）を行う
- [ ] 校正ラベル・識別を確認する
- [ ] おかしいと思ったら使用を止めて連絡（その器で測った品の扱いは品証が判断）★

関連場面: D2

## M5. 不良・ヒヤリの報告とトレーサビリティ
**目的**：小さな異常も上げる。責めない、仕組みで直す。

- [ ] 不良・ヒヤリ・ハットはためらわず報告する ★
- [ ] 個人を責めない前提で、データ・事実として共有する
- [ ] ロットと記録を紐づけて残す
- [ ] 週次で不良傾向を共有し、横展開のネタにする

関連場面: B3, B4, A2

## M6. 5S・現場管理と監査の日常準備
**考え方**：監査は特別なことでなく、日常の延長。日ごろの5Sと識別がそのまま備えになる。

- [ ] 5S（整理・整頓・清掃・清潔・躾）を日常で維持 ★
- [ ] 最新版の手順書・検査基準のみ現場に置く（旧版は回収）★
- [ ] 識別・表示（合格／保留／不良、ロット）を明確にする
- [ ] 記録をすぐ出せる状態にしておく

関連場面: D3, D5

## M7. 段取り替え・切替時の初物確認
**目的**：品種・ロット・材料の切替直後こそ不良が出やすい。初物で止める。

- [ ] 切替後の初物を確認してから連続生産に入る ★
- [ ] 図面・検査基準・限度見本が最新版か確認 ★
- [ ] 前品種の材料・仕掛りの混入防止（識別・清掃）
- [ ] 切替の記録（時刻・ロット・担当）を残す

関連場面: M3, D1, C1

## M8. 作業標準の順守と力量
**考え方**：決めたやり方でやる。できる人・できる工程を明確に。

- [ ] 最新の作業標準どおりに作業する（自己流にしない）★
- [ ] 認定された人が重要工程を担当する（無資格でやらない）★
- [ ] 分かりにくい・やりにくい手順は改善提案として上げる
- [ ] 教育・力量の記録を残す

関連場面: A5, D1, M1

# P. 購買向け

> **合言葉：「入口で止める。エビデンスで守る」**
> 購買（調達）は、社外から入る品質の“入口”。良いサプライヤーを選び、変化を管理し、エビデンスを集めることが、後工程のトラブルを防ぐ。品証と連携して、受入基準・是正・監査を回す。三本柱のひとつ「サプライヤーとの品質改善」の実働部隊。

## P1. サプライヤーの選定・承認
**目的**：新規の調達先は、品質の実力を確認してから採用する。

- [ ] 新規サプライヤーの品質評価（体制・認証・実績）を品証と実施 ★
- [ ] 初回サンプル・初品の妥当性を確認
- [ ] 品質保証条件（規格・検査・提出書類・変更連絡義務）を取り決め ★
- [ ] 承認の記録を残す

関連場面: C2, D3

## P2. 受入検査・受入基準
**目的**：何を・どの基準で受け入れるかを品証と決め、抜け漏れなく確認する。

- [ ] 受入検査基準（項目・抜取／全数・判定）を品証と設定 ★
- [ ] ミルシート・成績書の確認と保管
- [ ] 不適合時の処置ルール（受入保留・返品・特採の要否）
- [ ] トレーサビリティ（ロット・入荷日）を確保
