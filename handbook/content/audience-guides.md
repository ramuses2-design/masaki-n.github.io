# 読者別ハンドブック（連携ガイド）

> **位置づけ**：品質保証（QA）から、**営業・技術・製造・購買**の各部門、および**外注先（お取引先）**へ向けた連携の手引き。
> 「品証と、いつ・何を・どう連携するか」を場面別にまとめたもの。方針マスターの**ブレ防止6原則**と**エスカレーション基準**が土台。
> **考え方の核**：品質は検査で「見つける」より、**前工程で作り込む**。だから各部門・取引先との連携が品質の大半を決める。
> **凡例**：`★`＝特に外すと事故につながる重点項目。`【要確認】`＝GL（masaki）の確定が必要な箇所。各トピック末尾の「関連場面」から、品証内プレイブック（A〜D）へ辿れる。

## 読者別マップ
- **🤝 営業向け**：見積前確認／要求仕様の連携／クレーム一次対応／特採／出荷可否／監査対応／提出書類／仕様変更対応
- **🛠️ 技術向け**：新製品立ち上げ／4M変更／検査基準／設備導入／工程能力・データ活用／化学物質の設計織り込み／再発防止／治具・型管理
- **🏭 製造向け**：日常検査／異常時初動／変化点申告／測定器・校正／不良報告／5S・監査準備／切替初物／作業標準・力量
- **🧺 購買向け**：サプライヤー選定／受入検査／4M変更管理／化学物質エビデンス／不適合対応／監査・改善
- **📦 外注先向け**：求める品質の基本／提出書類／変更の事前連絡／不適合時の連絡／化学物質対応／納入・識別

# S. 営業向け

> **合言葉：「品質は“約束”。約束できないことは持ち帰る」**
> 営業は、お客様の声が最初に入る窓口。ここで拾う情報と、ここで交わす約束が、品質トラブルの起点にも、予防の起点にもなる。困ったら一人で抱えず、早めに品証（GL）へ一報を。

## S1. 引合い・見積前の品質確認
**目的**：受注前に「作れる・測れる・約束できる」を品証と確認し、無理な約束を防ぐ。

**確認して品証へ渡すこと**
- [ ] 要求仕様・規格・図面・適用法規を入手し品証へ共有 ★
- [ ] 用途・使用環境・お客様の重要特性（どこが効く品質か）を聞き取る
- [ ] 当社の実力（工程能力・検査可否・測定可否）を品証に確認 ★
- [ ] 特殊要求（清浄度・化学物質の非含有・トレーサビリティ・提出書類）の有無を確認
- [ ] 「品質面で約束できる範囲」を品証と握ってから見積る

**やってはいけない（NG）**
- 実力を確認せずに納期・スペックを口頭で約束する／「たぶん大丈夫」で受ける

関連場面: C2, C4

## S2. 顧客要求・仕様書の受け取りと連携
**目的**：お客様の要求を、抜け・取り違えなく品証が使える形で渡す。

- [ ] 仕様書・要求規格・検査要求・提出書類（成績書・非含有保証等）を一式で受領 ★
- [ ] 版数（改訂）を必ず最新で共有し、旧版と取り違えない ★
- [ ] 図面・規格の「特別特性」「重要保安部品」等の指定を見落とさない
- [ ] 受領した要求を品証へ連携し、対応可否・条件の回答を得る
- [ ] お客様固有の帳票・Webポータル・提出期限があれば品証と共有

関連場面: C4, D4, D5

## S3. クレーム・苦情の一次対応
**考え方**：「まず事実、約束は持ち帰る」。対外の最終窓口はGL／品証と連携して動く。

- [ ] 事実を聞き取る（現品・ロット・数量・発生状況・お客様の困りごと）★
- [ ] その場で原因・可否・賠償を約束しない ★
- [ ] 即、品証GLへ連携（クレーム・市場不良は**即エスカレーション**事象）★
- [ ] 暫定対応（代替品手配・選別）の要否を品証と決める
- [ ] 回答期限をお客様と握り、社内へ共有する

**やってはいけない（NG）**
- 憶測で原因を語る／確定していないことを約束する／独断で「無償対応します」と言う

関連場面: B2, B6, B1

## S4. 特採（特別採用）・仕様外品の取り扱い
**考え方**：規格外＝即NGではない。ただし**お客様承認なしに出荷可否を変えない**。

- [ ] 仕様外の可能性は品証へ相談（営業単独で判断しない）★
- [ ] 特採は必ずお客様の承認を取得する（無断で採用しない）★
- [ ] 安全・法規に関わるものは特採の対象外（絶対）★
- [ ] 適用範囲・数量・期限を限定し、恒久化させない
- [ ] 申請・承認・適用の記録を残す

関連場面: B5

## S5. 納期・出荷可否の約束の線引き
**目的**：品質判断（出荷可否）と営業判断（納期）の境界を守る。

- [ ] 出荷可否は、品質データに基づく品証の判断を待つ ★
- [ ] 品質保留中のロットを「間に合わせ」で出さない ★
- [ ] 短納期の依頼ほど早く品証へ共有（無理な短納期は品質リスク）
- [ ] GLが最終判断する観点（①信頼関係 ②売上影響 ③費用対効果）に関わる案件は必ずGLへ

関連場面: B1, A3

## S6. 監査・品質会議・顧客訪問での連携
**目的／考え方**：お客様の品質要求・評価を、社内へ正しく持ち帰る。監査・訪問は「見せる場」ではなく「相手の要求水準を知る場」。準備は日常の延長で足りるが、**日程だけは早く共有**しないと準備が間に合わない。

- [ ] 顧客監査・工場訪問の予定は早めに品証へ共有（準備に時間が要る）★
- [ ] 監査・訪問の**目的と範囲**（定期監査／新規採用の審査／クレーム後の確認）を事前に確認して伝える★
- [ ] 見学・撮影の可否、立入り範囲、同席者を事前に決める（当日その場で判断しない）★
- [ ] お客様の品質評価・要求の変化・競合情報を品証へフィードバック
- [ ] 品質会議の決定事項・宿題を記録し、期限を管理する
- [ ] その場で答えられない質問は、**推測で答えず「確認して回答する」**に統一する（期限を添える）★
- [ ] 当日の指摘・宿題は、社内の是正フローに載せて完了まで追う（口頭のまま終わらせない）★
- [ ] 【要確認】顧客監査時の立入り範囲・撮影可否・その場回答してよい範囲

**その場で答えない（持ち帰る）もの**
- 規格値・工程条件・原価に関わる数値、他社の情報、是正の完了時期の確約★

**やってはいけない（NG）**
- 監査日程を直前に共有し、準備を突貫でやらせる（普段どおりを見せるのが最短）
- その場の空気で対応可否や期限を約束する（S5・S8と同じ線引き）

関連場面: D3, A10, S7, S5

## S7. 提出書類・品質エビデンスの取り交わし
**目的**：お客様へ出す品質エビデンスを、正確・期限どおりに用意する。

- [ ] 必要書類（検査成績書・ミルシート・非含有保証・PPAP／初回流動）を品証と確認 ★
- [ ] 提出期限・提出先（帳票／Webポータル）を管理する
- [ ] 記載内容の最終確認（ロット・規格・測定値の整合）★
- [ ] 発行・提出の記録を残す（後で追える状態に）

関連場面: D5, S2

## S8. 顧客要求・仕様の変更対応
**考え方**：変更は「受けたら終わり」でなく、社内へ確実に展開して初めて完了。

- [ ] 変更（規格・図面・検査要求）を最新版で受領し版数を管理 ★
- [ ] 変更内容を品証・技術・製造へ速やかに展開 ★
- [ ] 対応可否・切替時期・在庫（旧仕様品）の扱いを社内と確認
- [ ] お客様へ切替時期・条件を回答する

関連場面: C1, C4, S2

## S9. クレーム後のフォロー・信頼回復
**考え方**：クレーム対応は「是正報告を出したら終わり」ではない。その後の作り込みと関係修復まで含めて“対応”。

- [ ] 是正報告（8D/CAPA）の内容をお客様目線でかみ砕いて説明★
- [ ] 恒久対策の効果を、次回以降の納入実績（データ）で示す★
- [ ] 再発ゼロの期間・改善件数など「良くなった事実」を可視化して共有
- [ ] お客様の要求水準の変化（監査強化・書類追加）を社内へ展開
- [ ] 一件の反省を横展開（同種製品・他顧客）して未然防止につなげる

関連場面: B4, B6, S3

## S10. 値上げ・価格改定交渉時の品質・仕様の整理
**考え方**：価格の話は品質・仕様とセット。何を（仕様・検査水準）どこまで（コスト影響）を整理してから交渉に臨む。

- [ ] 原材料・エネルギー・人件費など、コスト変動の根拠を事実で整理★
- [ ] 現行の仕様・検査水準・付帯サービス（検査成績書等）の棚卸し
- [ ] コスト最適化の代替案（仕様の見直し・数量まとめ）を技術と検討★
- [ ] 品質を落とさずに下げられる余地／落とせない一線を明確化
- [ ] 交渉で合意した仕様・条件は必ず書面化し、社内へ展開

関連場面: S2, P8, C1
関連場面: S2, P8, C1

## S11. 試作・サンプル対応と量産条件の橋渡し
**考え方**：試作品はゴールではなく入口。サンプルで合意した特性を、量産で再現できる条件に落とし込むまで見届ける。

- [ ] サンプル依頼時に、用途・評価項目・合否基準を顧客と明確化する★
- [ ] 試作条件（目付・加工・検査）を技術と記録し、量産との差を把握★
- [ ] 提出サンプルのロット・条件をトレースできるようにする
- [ ] 「サンプルはOK、量産で外れる」を防ぐため量産移行時の条件を技術と確認
- [ ] 量産化の可否・リードタイム・最小ロットを早めに社内で握る
- [ ] 【要確認】サンプル提出時の記録様式・量産移行判定の基準

関連場面: S2, T1, C2

## S12. 環境・サステナビリティ要求（脱炭素・リサイクル素材・エコ認証）への対応窓口
**考え方**：環境・サステナビリティ要求は「営業が受け、技術・購買・品証で答える」チーム対応。曖昧な口約束をせず、確認できる事実に基づいて回答する。

- [ ] 顧客の要求（再生材使用率・CO2・認証・有害物質不使用等）を具体的に文書で受け取る★
- [ ] 回答できる事実（エビデンスの有無）と、確認が必要な事項を切り分ける★
- [ ] リサイクル素材・認証（GRS等）は含有率・トレーサビリティの裏付けを技術・購買と確認
- [ ] バイオベース・生分解性素材への切替要求は、量産での品質妥当性（強度・風合い・ばらつき）が取れるまで数値・時期を約束しない（技術と条件出しを確認：TS29）★
- [ ] 化学物質・法規のエビデンスは既存の仕組み（SDS・chemSHERPA等）と連携（P4・D4）
- [ ] 数値・認証の約束は、社内で裏が取れたものだけを回答する（未確認は「確認します」）
- [ ] 【要確認】自社が提示できる環境エビデンスの範囲・認証の保有状況・回答窓口

関連場面: S2, P4, D4

## S13. 海外顧客・輸出対応（言語・規格・化学法規の差異）の窓口
**考え方**：海外案件は「国内と同じ品質保証を、言語・規格・法規の違いを翻訳して届ける」こと。曖昧な口約束をせず、確認できる事実に基づき、社内（技術・購買・品証）と連携して回答する。

- [ ] 顧客要求（適用規格・言語・化学法規＝REACH/化審法等・書類様式）を文書で受け取る★
- [ ] 国内基準との差異（試験方法・単位・許容値・必要エビデンス）を技術・品証と洗い出す★
- [ ] 化学物質・法規のエビデンスは既存の仕組み（SDS・chemSHERPA）と連携（P4・D4・T6）
- [ ] 輸送・通関・関税や納期の前提を購買・物流と確認し、約束の線引きを明確にする（S5）
- [ ] 数値・認証・法規適合の約束は、社内で裏が取れたものだけ回答する（未確認は「確認します」）
- [ ] 【要確認】自社が対応可能な海外規格・言語・輸出書類の範囲と回答窓口

関連場面: S2, S12, P4, D4

## S14. 相見積り・競合対応時の「品質価値」の伝え方（価格以外の訴求）
**目的／考え方**：価格勝負に流されず、当社の品質・トレーサビリティ・対応力を"見える価値"として顧客に伝え、値引き一辺倒の交渉を避ける。

- [ ] 顧客が本当に困っている品質・供給の不安（クレーム歴・法規対応・納期）を技術/品証と共有して把握する★
- [ ] 当社の強み（検査体制・トレーサビリティ・化学法規対応・改善実績）を具体的な根拠とともに整理する★
- [ ] 「安さ」ではなく「総コスト（不良・手直し・供給停止リスクの低減）」で価値を説明する
- [ ] 過度な品質保証・追加検査の口約束をその場でしない（原価・工数は品証/技術と確認）
- [ ] 失注時も理由を記録し、次の提案・改善に活かす
- [ ] 【要確認】訴求してよい実績・数値の公開範囲、価格決裁の権限

関連場面: S1, S10, A8

## S15. 納入仕様書・品質保証書の読み方（営業が最低限おさえる項目）
**目的／考え方**：納入仕様書・品質保証書は、営業がその場で完全に読み解く必要はない。ただし「顧客に約束してよい範囲」を見誤らないための最低限の読み方は身につけておく。判断に迷う記載は自己解釈で答えず、必ず品証・技術に確認してから回答する。

- [ ] 品名・型式・図面番号・版数が、受注情報・図面と一致しているか確認する★
- [ ] 保証している項目と、保証していない・対象外の項目（免責事項・使用条件の制限）を分けて読む★
- [ ] 適用規格・特性値の欄に意味が分からない専門用語や数値がある場合、そのまま顧客へ回答しない。必ず技術／品証に確認する★
- [ ] 版数・改訂日を確認し、古い版のまま社内保管・提出になっていないか確認する
- [ ] 発行元・発行日・押印など、書類として有効な体裁が整っているか確認する
- [ ] 顧客指定の様式（顧客フォーマットの品質保証書）で提出を求められた場合は、自己判断で記入せず品証と作成する★
- [ ] 【要確認】品証への確認が必須となる記載項目・営業が単独で回答してよい範囲の社内線引き

関連場面: S2, S1, T3

# T. 技術向け

> **合言葉：「前提が変わる＝新しいリスクが生まれる」**
> 技術（生産技術・開発）の設計・工程・設備の決定は、品質の作り込みそのもの。品証は「検査で止める前に、工程で作り込む」を掲げている。変更・立ち上げ・基準づくりを、品証と“先に”握るほど後戻りが減る。

## T1. 新製品・量産立ち上げ（APQP／PPAP 的）
**目的**：立ち上げ段階で品質を作り込み、量産後のクレームを防ぐ。

- [ ] 顧客要求・品質特性・規格を初期に明確化し品証と共有 ★
- [ ] 設計審査（DR）・FMEA的なリスク抽出に品証を参画させる ★
- [ ] QC工程表・検査基準・限度見本を品証と作成
- [ ] 初物確認・工程能力（Cp/Cpk）を確認してから量産移行 ★
- [ ] 顧客承認（PPAP／初回流動）の要否を確認
- [ ] 初期流動管理（立ち上げ直後の重点監視）を設定 ★

関連場面: C2, C4, D6

## T2. 4M変更管理（人・機械・材料・方法）
**考え方**：最大のリスクは“暗黙の変更”。**無断変更は禁止**。

- [ ] 人・機械・材料・方法・測定・環境の変更は、事前に品証へ ★
- [ ] 変更前に影響評価・リスクアセスメントを行う
- [ ] 顧客承認・4M変更届の要否を確認 ★
- [ ] 妥当性確認（初物・工程能力・試験）後に切り替える
- [ ] 変更履歴を記録する（いつ・何を・なぜ・誰が）

**やってはいけない（NG）**
- 「良くするだけだから」と、条件・材料・治具を無断で変える ★

関連場面: C1

## T3. 工程・検査基準の制定／改訂
**目的**：曖昧さのない、根拠のある基準を品証と合意する。

- [ ] 基準の根拠（規格・顧客要求・不具合履歴）を明確にする
- [ ] 測定方法・判定基準を曖昧さなく定義 ★
- [ ] 限度見本を整備（官能・外観検査は特に）
- [ ] 版管理（最新版のみ現場へ、旧版は回収）★

**回答が的外れなときの直し方（プロンプトを書き直す前に）**

**出力を検証する型（そのまま使わない）**
- [ ] 数値・固有名詞・日付・規格番号は、必ず一次情報（公式サイト・規格票・社内文書）と突き合わせる★

- [ ] 画面を見せて質問する使い方（Vision等）も、入力と同じ扱いになる。図面・規格・品質データ・顧客情報が映り込む画面では使わない★
- [ ] 判断に迷ったら「この画面・この文章を社外の人に見せてよいか」を基準にすると分かりやすい
- [ ] 使えるモデルや機能は更新で変わるが、入力してよい情報の範囲は変わらない（機能が増えても線引きは同じ）★

# M. 製造向け

> **合言葉：「異常はまず止める、それから直す」／「自己流にしない」**
> 品質は検査で見つける前に、工程で作り込む。現場の一つひとつの操作・記録・気づきが品質そのもの。難しい理屈より「これだけは守る」を優先。**悪い情報ほど早く**、ためらわず上げてほしい（責めるためではなく、仕組みで直すため）。

## M1. 日常の検査・検査表運用
**目的**：決めたとおりに測り、そのまま記録する。

- [ ] 始業前点検（測定器・標準・限度見本）★
- [ ] 検査基準どおりに実施する（自己流にしない）★
- [ ] 抜取／全数の区分を守る
- [ ] 記録は即時・正確に（後でまとめ書きしない）★
- [ ] ロット・日時・担当のトレーサビリティを残す

**やってはいけない（NG）**
- 「いつもと同じだから」で測定を省く／記録を後でまとめて埋める

関連場面: D1

## M2. 異常が出たときの初動（まず止める）
**合言葉**：まず止める、それから直す。一人で抱えない。

- [ ] 現品・現物を動かす前に記録／写真を撮る ★
- [ ] 止める・隔離する（赤札・識別で区別）★
- [ ] すぐ連絡する（上長・品証へ）★
- [ ] 勝手に手直し・選別して流さない ★
- [ ] いつから・どれだけ（数量・期間）を分かる範囲で伝える

関連場面: B2, B1

## M3. 4M変化点の申告（無断で変えない）
**考え方**：いつもと違うことは、良いことでも必ず申告。

- [ ] 材料ロット替え・設備調整・人の交代・方法変更は事前に申告 ★
- [ ] 「良くするつもりの変更」も無断でやらない ★
- [ ] 変化点があった日は記録に残す
- [ ] 異常の芽（音・におい・ばらつき・手ごたえの違い）に気づいたら共有する

関連場面: C1, B3

## M4. 測定器・校正の順守
**目的／考え方**：「正しく測れる道具」でしか判定しない。測定器が狂っていると、**不良を良品と判定して流出**する。しかも記録上は「合格」なので、後から気づきにくい。だから測る前に道具を確認する。

- [ ] 校正期限切れの測定器を使わない ★
- [ ] 使用前点検（ゼロ点・ばらつき）を行う
- [ ] 校正ラベル・識別を確認する（ラベルが無い・読めない器は使わない）★
- [ ] 落とした・ぶつけた・水濡れした測定器は、期限内でも使用を止めて連絡する★
- [ ] おかしいと思ったら使用を止めて連絡（その器で測った品の扱いは品証が判断）★
- [ ] 自分で調整・分解しない（「直したつもり」が一番危ない）★

**校正切れ・異常が分かったときは「さかのぼる」**
- [ ] その測定器で**前回の正常確認以降に測ったロット**を洗い出す（測った本人が分かるうちに）★
- [ ] 該当ロットの出荷状況を確認し、出荷済みがあれば即・品証へ（B1のトリアージへ）★
- [ ] 「今日から新しい器で測る」だけで終わらせない。過去の判定が有効かどうかは品証が判断する★

**やってはいけない（NG）**
- 校正期限が「もうすぐ切れる」器を、切れるまで使い切る
- 期限切れに気づいたが、測った結果は問題なかったので黙っておく★

関連場面: D2, M13, B2

## M5. 不良・ヒヤリの報告とトレーサビリティ
**目的**：小さな異常も上げる。責めない、仕組みで直す。

- [ ] 不良・ヒヤリ・ハットはためらわず報告する ★
- [ ] 個人を責めない前提で、データ・事実として共有する
- [ ] ロットと記録を紐づけて残す
- [ ] 週次で不良傾向を共有し、横展開のネタにする

関連場面: B3, B4, A2

## M6. 5S・現場管理と監査の日常準備
**考え方**：監査は特別なことでなく、日常の延長。日ごろの5Sと識別がそのまま備えになる。

- [ ] 5S（整理・整頓・清掃・清潔・躾）を日常で維持 ★
- [ ] 最新版の手順書・検査基準のみ現場に置く（旧版は回収）★
- [ ] 識別・表示（合格／保留／不良、ロット）を明確にする
- [ ] 記録をすぐ出せる状態にしておく

関連場面: D3, D5, D8

## M7. 段取り替え・切替時の初物確認
**目的**：品種・ロット・材料の切替直後こそ不良が出やすい。初物で止める。

- [ ] 切替後の初物を確認してから連続生産に入る ★
- [ ] 図面・検査基準・限度見本が最新版か確認 ★
- [ ] 前品種の材料・仕掛りの混入防止（識別・清掃）
- [ ] 切替の記録（時刻・ロット・担当）を残す

関連場面: M3, D1, C1

## M8. 作業標準の順守と力量
**考え方**：決めたやり方でやる。できる人・できる工程を明確に。

- [ ] 最新の作業標準どおりに作業する（自己流にしない）★
- [ ] 認定された人が重要工程を担当する（無資格でやらない）★
- [ ] 分かりにくい・やりにくい手順は改善提案として上げる
- [ ] 教育・力量の記録を残す

関連場面: A5, D1, M1

## M9. 多能工化と応援時の品質担保
**考え方**：応援・多能工はありがたいが、力量が伴わないと不良の入口。「できる範囲」を見える化して任せる。

- [ ] 各工程に必要な力量（認定）を明確化し、応援者の可否を判断★
- [ ] 初めての工程は、作業標準の確認と初物チェックをセットで★
- [ ] 応援も4M変化点として申告（無断で人を替えない）
- [ ] 難所・不良の出やすい点を口頭でなく標準・写真で伝える
- [ ] 応援後は結果（不良・ヒヤリ）を振り返り、力量表を更新

関連場面: M3, M8, A5

## M10. ヒューマンエラー防止（ポカヨケ・指差呼称・ダブルチェック）
**考え方**：「気をつける」だけに頼らない。間違えられない仕組み（ポカヨケ）で品質を守る。

- [ ] 取り違え・付け忘れ・数量ミスが起きやすい工程を洗い出す★
- [ ] 治具・色分け・表示・システムチェックで“物理的に防ぐ”工夫を入れる★
- [ ] 重要工程は指差呼称・ダブルチェックを標準作業に組み込む
- [ ] 発生したエラーは責めずに記録し、仕組みの改善につなげる
- [ ] 【要確認】重要管理点（取り違え厳禁工程）とポカヨケの標準

関連場面: M2, M7, D5
関連場面: M2, M7, D5

## M11. 交替勤務・引き継ぎ時の品質情報の伝達
**考え方**：品質のトラブルは「言った・言わない」で起きやすい。交替のたびに“異常・変化点・申し送り”を確実に渡す。

- [ ] 申し送り事項（異常・変化点・特採・注意ロット）を定型の帳票で残す★
- [ ] 進行中の異常対応・隔離品の状態を次直へ確実に引き継ぐ★
- [ ] 設備・測定器の異常やヒヤリは口頭だけでなく記録で共有
- [ ] 引き継ぎ漏れが起きた事例は責めずに帳票・ルールを改善する
- [ ] 【要確認】申し送り帳票の様式・必須記載項目・確認サインの運用

関連場面: M2, M5, D5

---

## M12. 現場の温湿度・環境管理（吸湿・静電気・異物の予防）
**考え方**：不織布は環境（温度・湿度・清浄度）の影響を受けやすい。現場の環境を安定させることが、寸法・静電気・異物・カビの予防につながる。

- [ ] 定められた区域の温度・湿度を定期記録し、基準を外れたら申告する★
- [ ] 原反・製品の吸湿を防ぐため、開梱・仕掛品の放置ルールを守る（TS17と連動）★
- [ ] 乾燥期は静電気・異物付着に注意し、除電・清掃の頻度を上げる（TS14と連動）
- [ ] 結露・カビの兆候（におい・変色）を見つけたら隔離して報告する
- [ ] 空調・除湿・除電設備の異常は放置せず保全・上長へ連絡する
- [ ] 【要確認】区域ごとの温湿度基準・記録様式・逸脱時の対応手順

関連場面: M2, M6, D5

---

## M13. 始業前点検・設備コンディション確認（チョコ停・品質異常の予防）
**目的**：品質異常の多くは「いつもと違う」設備状態から始まる。始業前の短い点検で、チョコ停や不良の芽を稼働前につぶす。

- [ ] 始業前チェック項目（温度・圧力・エア・ロール表面・異音・付着物）を確認する★
- [ ] 前直の引き継ぎ（異常・注意ロット・調整履歴）を確認する（M11）★
- [ ] 初物確認・初品検査を行い、規格内であることを確認してから量産に入る（M7）
- [ ] 「いつもと違う」兆候（音・振動・温度・くず量）は放置せず記録・報告する（M2）
- [ ] 点検で見つけた不具合は当日中の処置と保全連携につなげる（T9）
- [ ] 【要確認】始業前点検表の項目・判定基準・対象設備

関連場面: M2, M7, M11

---

## M14. 限度見本・現物基準の現場運用（保管・更新・照合）
**目的／考え方**：目視検査の「合否の境目」を人によってブレさせない。限度見本を正しく保管・更新し、日々の検査で照合できる状態を保つ。

- [ ] 使用中の限度見本が最新版か（有効期限・改訂番号）を始業時に確認する★
- [ ] 見本の劣化（変色・汚れ・破損）がないか定期点検し、劣化時は差し替える★
- [ ] 見本は決められた場所・条件で保管し、私物化・持ち出しを防ぐ
- [ ] 判定に迷う現物は自己判断せず、リーダー・品証にエスカレーションする
- [ ] 見本の改訂・廃棄は記録を残し、旧版は現場から回収する
- [ ] 【要確認】限度見本の管理台帳・更新周期・保管場所、改訂の承認者

関連場面: M1, TS6, B1

## M15. 改善提案・気づきの出し方（小さな異常を言葉にする）
**目的／考え方**：大きな不良は、たいてい「なんとなく変」を誰も口に出さなかったところから始まる。うまく言えなくていいので、気づいた時点で出すことが最大の予防になる。

- [ ] 「いつもと違う」と感じたら、原因が分からなくてもその場で口に出す・書き留める（原因の特定は自分の仕事ではない）★
- [ ] 出すときは「いつ・どの設備/ロット・何がどう違ったか」の3点だけ添える（きれいな文章にする必要はない）★
- [ ] 「自分だけかも」「今さら言いにくい」は理由にしない。空振りの報告は歓迎される（見逃しの方がはるかに高くつく）
- [ ] 改善提案は、まず自分の手元の「毎日ちょっと面倒な作業」から選ぶ（大きなテーマでなくてよい）
- [ ] 提案が採用されなかった場合も、理由を聞いておくと次の精度が上がる
- [ ] 【要確認】気づき・提案の提出先と様式、フィードバックの返し方、表彰・工数の扱い

関連場面: A6, M2, M5, M13

## M16. 検査記録の書き方・訂正ルール（二重線・日付・空欄禁止）
**目的／考え方**：検査記録は「後から証明できる記録」であって初めて意味を持つ。書き方・訂正の仕方に自己流を持ち込まない。

- [ ] 記録は検査したその場で記入する（後でまとめて書かない・記憶で埋めない）★
- [ ] 空欄を作らない。該当なしの項目は所定の記号（「-」等）で明示的に埋める★
- [ ] 誤記の訂正は修正液・塗りつぶしを使わず、二重線＋訂正印（またはサイン）＋訂正日で行う★
- [ ] 数値は決められた単位・桁数で記入する（丸め方を自己判断しない）
- [ ] 記入者・確認者の欄は本人が記入する（代筆・後日まとめ書きをしない）★
- [ ] 異常値・規格外を確認したら、記録を消さずにそのまま残し、B2（初動・封じ込め）の手順に従って報告する
- [ ] 【要確認】検査記録の保管期間・保管場所、電子化している場合の訂正ルール

関連場面: M1, D5, B2

# P. 購買向け

> **合言葉：「入口で止める。エビデンスで守る」**
> 購買（調達）は、社外から入る品質の“入口”。良いサプライヤーを選び、変化を管理し、エビデンスを集めることが、後工程のトラブルを防ぐ。品証と連携して、受入基準・是正・監査を回す。三本柱のひとつ「サプライヤーとの品質改善」の実働部隊。

## P1. サプライヤーの選定・承認
**目的**：新規の調達先は、品質の実力を確認してから採用する。

- [ ] 新規サプライヤーの品質評価（体制・認証・実績）を品証と実施 ★
- [ ] 初回サンプル・初品の妥当性を確認
- [ ] 品質保証条件（規格・検査・提出書類・変更連絡義務）を取り決め ★
- [ ] 承認の記録を残す

関連場面: C2, D3

## P2. 受入検査・受入基準
**目的**：何を・どの基準で受け入れるかを品証と決め、抜け漏れなく確認する。

- [ ] 受入検査基準（項目・抜取／全数・判定）を品証と設定 ★
- [ ] ミルシート・成績書の確認と保管
- [ ] 不適合時の処置ルール（受入保留・返品・特採の要否）
- [ ] トレーサビリティ（ロット・入荷日）を確保

関連場面: D1, B5, P10

## P3. サプライヤーの4M変更・製造中止管理
**考え方**：仕入品の“暗黙の変更”は自社クレームの主因。事前連絡を義務化する。

- [ ] サプライヤーに4M変更の事前連絡を求める（契約・覚書に明記）★
- [ ] 変更時は初品・妥当性を再確認 ★
- [ ] 製造中止（EOL）・材料変更の早期把握と代替検討
- [ ] 変更履歴を記録する

関連場面: C1, D4

## P4. 化学物質・法規エビデンスの収集
**目的**：顧客要求・法規に応えるため、成分情報を確実に集める。

- [ ] SDS・chemSHERPA・IMDS・非含有保証をサプライヤーから収集 ★
- [ ] 顧客要求基準・適用法規（RoHS／REACH）との突合 ★
- [ ] 材料・工程変更時の再取得（変更管理と連動）
- [ ] 記録・トレーサビリティを残す

関連場面: D4, C1

## P5. サプライヤー不適合・クレーム対応
**目的**：仕入品の不具合を、是正まで確実に回す。

- [ ] 不適合の封じ込め（受入停止・在庫隔離）★
- [ ] サプライヤーへ是正要求（原因・恒久対策・横展開／8D）★
- [ ] 是正の期限管理と効果確認
- [ ] スコア・評価へ反映（継続取引の判断材料に）

関連場面: B2, B4

## P6. サプライヤー監査・品質改善の伴走
**考え方**：選ぶだけでなく、育てる。三本柱「サプライヤー品質改善」の実務。

- [ ] 定期監査（品質体制・工程・記録）を計画・実施 ★
- [ ] 課題の共有と改善の伴走
- [ ] 重要サプライヤーの複数社化・BCPの検討
- [ ] 監査・改善の結果をマネジメントレビューへ

関連場面: D3, D6, A7, P5

## P7. 供給BCP・二社購買（供給途絶リスク）
**考え方**：品質が良くても「入らない」と止まる。重要材料は供給途絶リスクを品質と同じ土俵で管理する。

- [ ] 重要材料・単一供給（シングルソース）品目を棚卸し★
- [ ] 二社購買・代替材の可否を技術・品証と検討（同等性評価）★
- [ ] 供給業者のBCP・在庫方針・リードタイムを把握
- [ ] 災害・需給逼迫時の優先供給・連絡ルートを事前に取り決め
- [ ] 代替材の採用は4M変更として品質妥当性を確認（無条件切替は不可）
- [ ] 供給途絶は「災害」だけではない。取引先の統合・買収・事業売却・拠点集約でも、供給条件と品質保証体制は動く★
- [ ] 重要材料の供給元について、業界再編の兆候（統合・スピンオフ・工場閉鎖の公表）を掴んだら、その時点で影響評価を始める（TS30）★
- [ ] 【要確認】重要材料リストと最低在庫・安全在庫の基準

関連場面: P1, P3, C1, TS30

## P8. 原材料の値上げ・代替材の評価と切替管理
**考え方**：コスト・供給リスク対応で材料を変える時こそ、品質と法規の確認を飛ばさない。

- [ ] 値上げ／供給不安の材料をリスト化し、影響度で優先順位づけ★
- [ ] 代替材の候補について、規格・特性・法規（SDS/含有情報）を確認★
- [ ] 少量での評価（受入・工程・製品特性）を技術と連携して実施
- [ ] 4M変更として正式に管理し、切替時期・在庫の入れ替えを計画
- [ ] 【要確認】代替材の承認基準・評価項目・顧客承認の要否

関連場面: P1, P3, C1

## P9. 受入ロット管理・FIFO と保管品質の維持
**考え方**：良い材料を買っても、保管で劣化すれば台無し。ロットを識別し、先入れ先出しと保管環境で品質を守る。

- [ ] 受入時にロット番号・数量・試験成績書を記録し識別する★
- [ ] 先入れ先出し（FIFO）を徹底し、在庫の滞留・期限切れを防ぐ★
- [ ] 温湿度・直射・においなど、材料ごとの保管条件を守る
- [ ] 使用期限・再評価の要否を管理し、期限切れ材料の混入を防ぐ
- [ ] トレーサビリティのため、材料ロットと製造ロットの紐付けを維持
- [ ] 【要確認】材料別の保管条件・使用期限・受入記録の様式

関連場面: P2, D4, C1

---

## P10. 副資材・包装資材の品質管理（インク・粘着・段ボール等の変更管理）
**考え方**：主原料だけでなく、インク・粘着剤・離型紙・段ボール等の副資材も品質・法規に影響する。「副資材だから」と変更管理を緩めない。

- [ ] 副資材も含有化学物質・法規（SDS等）のエビデンスを収集・保管する★
- [ ] 副資材メーカーの材質・製法変更（4M変更）を事前連絡させる仕組みを作る★
- [ ] 印刷インク・粘着剤の変更は、製品側の品質（付着・臭気・移行）への影響を技術と確認
- [ ] 包装（段ボール・フィルム）の変更は輸送ダメージ・結露への影響を確認（TS15と連動）
- [ ] 副資材の受入・保管条件（吸湿・期限）を主原料に準じて管理する
- [ ] 【要確認】副資材の受入基準・変更連絡ルール・対象副資材のリスト

関連場面: P3, P4, TS15

---

## P11. 輸入材・調達リスク管理（為替・関税・長納期の変動対応）
**目的**：円安・関税・海運の変動は、原材料のコストと供給の両方を揺らす。価格転嫁と供給途絶の両リスクを、品質を落とさずに管理する。

- [ ] 輸入材・海外依存度の高い品目を洗い出し、為替・関税変動の影響度を把握する★
- [ ] 長納期品・単一調達品はリードタイムと安全在庫を見直す（P7・P9）★
- [ ] 代替材・国内代替先の候補を評価し、切替時は品質同等性を技術・品証と確認（P8）
- [ ] 価格改定・条件変更はサプライヤーの4M/製造中止情報と合わせて管理（P3）
- [ ] コスト変動の見通しは営業の価格交渉（S10）と共有し、無理な在庫積み増しを避ける
- [ ] 【要確認】重点輸入品目・安全在庫水準・代替先の承認状況

**為替前提を「書いて」おく（後から揉めないために）**
- [ ] 見積書・価格改定の資料には、前提とした為替水準と有効期限を明記する。前提を書かない見積りは、変動時に値上げ交渉の根拠を失う★
- [ ] 円安局面では輸入材のコスト増と、代替材への切替圧力が同時に来る。切替は必ず品質妥当性の確認とセットにし、コスト都合だけで進めない（P8・TS29）★
- [ ] 為替の水準そのものは日々動く。ハンドブックには数値を書かず、都度の最新値を前提として明示する運用にする

関連場面: P7, P8, P3, TS29

---

## P12. 受入データの活用・サプライヤー品質スコアリング（受入不良率の見える化）
**目的／考え方**：受入検査やクレームのデータを溜めるだけにせず、サプライヤーごとに品質を数値化して、改善要請・二社購買・承認見直しの判断に使う。

- [ ] サプライヤー別に受入不良率（PPM）・不適合件数・是正対応の速さを定期集計する★
- [ ] 悪化傾向のサプライヤーを早期に把握し、改善要請・監査(P6)につなげる★
- [ ] スコアは技術・品証と共有し、承認継続/二社購買(P7)の判断材料にする
- [ ] 良好なサプライヤーは受入検査の頻度緩和など合理化の対象として検討する
- [ ] データの取り方（不適合の定義・集計期間）を揃え、恣意的な評価を避ける
- [ ] 【要確認】スコアの評価基準・見直し頻度・共有範囲、承認見直しの権限
