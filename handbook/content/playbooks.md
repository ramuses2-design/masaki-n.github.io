# 品質保証グループ 場面別プレイブック集（網羅版）

> **位置づけ**：方針マスター（考え方・ルール）を「いつ・どの場面で・何をするか」に落とした**行動指針**。
> 各場面の手順とチェック項目に沿えば、やり方に漏れが出ない状態を目指す。
> **凡例**：`【要確認】`＝あなた（GL）の確定が必要な箇所。`★`＝特に外すと事故につながる重点項目。
> **共通の型**：① 事実確認 → ② 影響評価 → ③ 打ち手 → ④ 記録 → ⑤ 横展開（どの場面でも同じ順番で回す）。

## 場面マップ（4グループ）
- **A 定例・マネジメント**：A1 1on1／A2 週次／A3 月次／A4 目標設定・評価／A5 教育・力量管理・新任受け入れ
- **B 異常対応**：B1 トリアージ／B2 初動・封じ込め／B3 原因究明／B4 是正・恒久・横展開／B5 特採申請／B6 顧客連絡・対外／B7 重大品質問題・リコール級
- **C 変化・プロジェクト**：C1 4M変更管理／C2 新製品・量産立ち上げ／C3 設備導入・改善／C4 工程・検査基準の制定改訂
- **D 予防・維持・システム**：D1 日常検査・検査表運用／D2 測定器・校正・MSA／D3 監査対応／D4 化学物質・法規／D5 文書・記録管理／D6 KPI・品質目標

# A. 定例・マネジメント

## A1. 1on1（育成・面談）
**目的**：育成・負荷把握・状態確認の双方向の場。進捗の詰問・報告会にしない。

**準備（GL）**
- [ ] 前回のアクション・約束を確認しておく
- [ ] 本人の最近の実績・負荷データ（工数・不良対応）に目を通す
- [ ] 今日のテーマを1つ用意（毎回“育成の問い”を持つ）

**進め方（傾聴7割）**
- [ ] ①コンディション（体調・負荷・困りごと）をまず聞く
- [ ] ②業務の障害（自分では外せない壁）を引き出す → GLが外す
- [ ] ③成長（スキルマップの現在地と次の一歩・任せたい役割）
- [ ] ④改善テーマの進み（改善の時間が取れているか）
- [ ] ⑤キャリア・要望（中期で挑戦したいこと）
- [ ] 最後に「次アクションと期限」を一緒に決める

**GLが必ず確認（観察）★**
- [ ] 過負荷・疲弊の兆候（残業・表情・口数）
- [ ] 属人化して休めない業務はないか（副担当の要否）
- [ ] 改善・学習の時間が取れているか
- [ ] 安全・コンプライアンス・ハラスメントの懸念
- [ ] モチベーション低下のサイン

**やらないこと（NG）**
- 進捗報告だけで終える／一方的に指示して終える／不満を放置する／評価面談と混同する

> 【要確認】頻度（例：月1回30分）・記録の残し方。

## A2. 週次ミーティング（グループ内）
**目的**：異常の共有と、月次に出す数字・メッセージを先に揃える（伝達のブレ防止）。

**アジェンダ**
- [ ] 異常・不適合・ヒヤリの共有（先週発生分）★
- [ ] 進捗（計画対実績）と未達の要因
- [ ] 不良傾向（パレート・推移）
- [ ] 負荷状況（偏り・属人化のサイン）
- [ ] 改善テーマの進捗（5〜10%が機能しているか）
- [ ] 来週の重点と役割分担
- [ ] 月次に出す数字・結論の確定★

**確認チェック**
- [ ] 異常・不適合の上げ漏れはないか
- [ ] 即エスカレーションすべき案件はないか
- [ ] 負荷の偏り・属人化の再配分は要るか
- [ ] 先週のアクションはクローズしたか
- [ ] 月次の数字を全員が同じ言葉で言えるか

## A3. 月次報告（部門横断：製造係長＋他GL＋事業部長）
**目的**：意思決定に必要なことを、結論ファーストで上げる。

**報告の型（順番）**
1. 結論（今月、品質はどうか・何を決めてほしいか）
2. 数字（KPI：不良率・クレーム/不適合件数・是正の状況・改善進捗）
3. 重要案件（顧客・監査・重大不適合の状況と対応）
4. 改善ロードマップの進捗（アプリ移行・検査表標準化 等）
5. 依頼・エスカレーション（誰が・いつまでに）

**準備チェック**
- [ ] 数字は週次でグループ内と一致しているか★
- [ ] 悪い情報を先に出す構成か（後回しにしない）
- [ ] 依頼は「誰が・いつまでに」まで言えるか
- [ ] 前月の依頼・宿題のクローズ状況を示せるか

> 【要確認】月次KPIの正式定義・目標値・フォーマット。

## A4. 目標設定・評価（期初／期中／期末）
**目的**：組織目標を個人に接続し、事実に基づいて公正に評価・育成する。

**期初（目標設定）**
- [ ] 事業部・グループ目標を個人目標へブレイクダウン
- [ ] SMART（具体・測定可能・達成可能・関連・期限）で設定
- [ ] 「業務目標」＋「スキル目標」＋「改善目標」をセットで
- [ ] 評価軸・期待役割を本人と共有（後出しにしない）★

**期中（モニタ）**
- [ ] 1on1で進捗を確認、つまずきを早期に支援
- [ ] 環境変化があれば目標を見直す（固定しない）

**期末（評価）**
- [ ] 事実・データで評価する（印象で決めない）★
- [ ] 本人の自己評価とすり合わせる
- [ ] フィードバックは行動ベースで具体的に
- [ ] 良い点と次の課題を両方伝え、次期へつなぐ
- [ ] 評価基準は全員に一貫しているか（えこひいきの排除）

> 【要確認】評価軸の中身・評価サイクル（半期/通期）・処遇との連動。

## A5. 教育・力量管理／新任受け入れ
**目的**：必要な力量を定義し、計画的に育て、記録する（ISO の力量管理）。

**スキルマップ運用**
- [ ] 業務ごとに必要力量レベルを定義
- [ ] 各人の現在地を評価しギャップを可視化
- [ ] ギャップに教育・OJT計画を割り当て
- [ ] 力量の認定・記録を残す★
- [ ] 重要工程は認定者のみが担当（無資格者にやらせない）★

**新任受け入れ（オンボーディング）**
- [ ] ①安全教育 → ②品質の基本・ルール → ③OJT（副担当を付与）
- [ ] 独り立ち判定の基準を事前に決める
- [ ] 一定期間後にフォロー面談（定着確認）
- [ ] 教育記録を残す

> 【要確認】スキルマップの項目・力量認定の基準・新任の標準育成期間。

# B. 異常対応（不適合・クレーム）

> 合言葉＝**「まず止める、それから直す」**。スピードと優先順位がすべて。一人で抱えない。

## B1. 優先順位（トリアージ：上から確認）
1. **人の安全**：人身・労災・安全に関わるか ★
2. **法規制・コンプラ**：法令・規制（化学物質・安全基準等）違反の恐れ ★
3. **顧客への影響**：ライン停止・出荷済み・市場流出の恐れ ★
4. **流出範囲**：社内でどこまで波及（ロット・数量・期間）
5. **社内損失**：手直し・廃棄・納期影響

> 1〜3 のいずれかに当たれば **即・GLへ直接**（メール／Teams／電話／口頭）。

## B2. 初動・封じ込め（流出を止める）
- [ ] **現品・現物の確保**（不良品・現物・ラベル・ロットを動かす前に記録／写真）★
- [ ] **封じ込め**：出荷停止・在庫の隔離・該当品の識別（赤札等）・仕掛品の確認★
- [ ] **影響範囲の特定**：ロットトレース（前後ロット・同条件品・他顧客・他品種への波及）
- [ ] **数量・期間の特定**（いつから・どれだけ）
- [ ] **暫定（応急）対策**：全数検査・選別・代替品手配で安全側へ
- [ ] **顧客への一次連絡**：事実・暫定対応・調査予定・回答期限（対外はGL／窓口）★
- [ ] **社内連携**：製造・営業・関係部門へ即共有

## B3. 原因究明（発生と流出の両面）
- [ ] 発生メカニズムを特定（どこで・なぜ作り込まれたか）
- [ ] なぜなぜ分析（5回）で真因まで掘る
- [ ] **4M（人・機械・材料・方法）＋変化点**で切る★
- [ ] **流出原因も別に分析**（なぜ検査で止まらなかったか）★
- [ ] 再現性を確認（再現しなければ仮説を疑う）
- [ ] データ・事実で裏づける
- [ ] 個人の責任に帰結させず「仕組み」で捉える★

## B4. 是正処置・恒久対策・横展開（8D／CAPA）
- [ ] 恒久対策を「**発生対策＋流出対策**」の両面で立てる★
- [ ] 効果を検証（対策後のデータ・一定期間で確認）
- [ ] 標準化：手順書・検査基準・QC工程表へ反映★
- [ ] 教育して定着させる
- [ ] **横展開**：同等品・他ライン・他工程へ水平展開★
- [ ] 完了判定とフォロー（再発していないか）
- [ ] 是正処置報告書（8D等）として記録
- [ ] 顧客への是正回答を期限内に提出

> 【要確認】是正処置の正式様式（8D/CAPA）・完了判定の責任者。

## B5. 特採（特別採用）申請の判断
**考え方**：規格外＝即NGではない。ただし安全・法規は絶対に妥協しない。常態化させない。

- [ ] **安全・法規に関わるものは特採しない**（絶対）★
- [ ] お客様の用途・機能に影響しないかを確認
- [ ] **顧客の承認が必須**（無断で出荷可否を変えない）★
- [ ] 根拠データを添付（規格・試験・実績）
- [ ] 適用範囲・数量・期限を限定（恒久措置にしない）
- [ ] 出荷可否の最終判断はGL
- [ ] 恒久対策（規格外を出さない仕組み）とセットで進める
- [ ] 申請・承認・適用を記録

## B6. 顧客連絡・対外対応の作法
- [ ] 窓口はGL／対外担当（現場が直接抱え込まない）★
- [ ] 一次連絡は早く（事実・暫定対応・調査予定・回答期限）
- [ ] 憶測で語らない／確定していないことを約束しない★
- [ ] 約束は守れる範囲で、守れる期限で
- [ ] 是正回答の期限を管理（遅れそうなら事前連絡）
- [ ] 報告書の品質を担保（事実・原因・対策・再発防止）
- [ ] 報告後の関係維持・信頼回復まで見る

## B7. 重大品質問題・リコール級のエスカレーション
**トリガー**：人身・安全、法規違反、広範な市場流出、ブランド毀損の恐れ。

- [ ] **即、事業部長・経営へ報告**（時間を置かない）★
- [ ] 安全・法規を最優先で対処
- [ ] 出荷停止・回収範囲の特定（市場・在庫・仕掛）
- [ ] 顧客・規制当局への報告要否を確認★
- [ ] 対応体制（対策チーム）を立てる
- [ ] 記録を徹底（時系列・判断・対応）
- [ ] 恒久対策と外部への正式報告／再発防止

> 【要確認】重大問題の社内定義・連絡網・対策本部の起動基準。

# C. 変化・プロジェクト

> 考え方＝**「前提が変わる＝新しいリスクが生まれる」**。変化は無断で起こさない。

## C1. 4M変更管理（人・機械・材料・方法）
**最大のリスクは“暗黙の変更”**。気づかぬ変更がクレームの主因になる。

- [ ] 変更の検知（材料・設備・工法・人・測定・環境のどれか）★
- [ ] 変更前に影響評価とリスクアセスメント
- [ ] **顧客承認の要否を確認**（契約・仕様・4M変更届）★
- [ ] 妥当性確認（初物・工程能力・試験）
- [ ] 変更履歴を記録（いつ・何を・なぜ・誰が）
- [ ] 関係者へ周知・教育
- [ ] 変更後フォロー（初期流動で異常がないか）
- [ ] **無断変更の禁止を全員に徹底**★

> 【要確認】顧客承認が必要になる変更の社内線引き。

## C2. 新製品・量産立ち上げ（APQP／PPAP 的）
- [ ] 要求仕様・顧客要求を明確化（品質特性・規格）
- [ ] 設計審査（DR）に参画しリスクを抽出（FMEA的視点）
- [ ] 工程設計・QC工程表（管理項目・管理水準・検査）
- [ ] 検査基準・限度見本の作成
- [ ] 設備・治具・測定器の準備と妥当性（校正・MSA）
- [ ] 初物確認・工程能力（Cp/Cpk）の確認★
- [ ] 必要に応じて顧客承認（PPAP／初回流動）★
- [ ] 標準書・教育の整備
- [ ] 量産移行判定（基準を満たすか）
- [ ] 初期流動管理（立ち上げ直後の重点監視）★

## C3-a. 設備の導入・改善：企画・安全
- [ ] 目的・解決する品質課題が明文化されているか★
- [ ] 要求仕様（精度・能力・タクト）が明確か
- [ ] 費用対効果を確認（GL判断軸：信頼関係・売上影響・費用対効果）
- [ ] リスクアセスメント／残留リスクの対策★
- [ ] 関連法規・社内安全基準（労安・電気/PSE 等）
- [ ] 非常停止・ロックアウト・安全カバー
- [ ] 環境影響（化学物質・排気排水・廃棄物：ISO14001）
- [ ] 据付環境・動力・レイアウト

## C3-b. 設備の導入・改善：品質妥当性・維持
- [ ] 据付確認・稼働確認・性能確認（IQ/OQ/PQ 的）★
- [ ] 測定設備なら校正・MSA（Ga R&R）で「測れる」を証明★
- [ ] 初物・工程能力（Cp/Cpk）の確認
- [ ] 4M変更管理：顧客承認・届出の要否
- [ ] 保全計画・予備品・点検基準の整備
- [ ] 作業手順書・教育（副担当含む／属人化しない）★
- [ ] 導入後の効果検証（before/after）
- [ ] 横展開先の洗い出し

> 【要確認】設備投資の決裁フロー。

## C4. 工程・検査基準の制定／改訂
- [ ] 基準の根拠を明確に（規格・顧客要求・不具合履歴）
- [ ] 測定方法・判定基準を曖昧さなく定義★
- [ ] 限度見本を整備（官能検査は特に）
- [ ] ばらつき・測定の再現性を考慮
- [ ] 文書化と版管理（最新版のみ現場へ）★
- [ ] 関係者へ教育・周知
- [ ] 改訂時は影響評価と顧客承認の要否を確認★
- [ ] 旧版の回収（混在防止）

# D. 予防・維持・システム

## D1. 日常の検査・検査表運用
- [ ] 始業前点検（測定器・標準・限度見本）
- [ ] 検査基準どおりに実施（自己流にしない）★
- [ ] 抜取／全数の区分を守る
- [ ] 記録は即時・正確に（後でまとめ書きしない）★
- [ ] 異常時の処置ルール（止める・隔離・連絡）を守る★
- [ ] トレーサビリティ（ロット・日時・担当）
- [ ] ダブルチェックの要否を明確化
- [ ] 検査表の保管・提出

## D2. 測定器・校正・MSA管理
- [ ] 校正計画と有効期限の管理（期限切れ使用ゼロ）★
- [ ] 校正ラベル・識別
- [ ] 社内校正／外部校正の区分と基準器のトレーサビリティ
- [ ] 使用前点検（ゼロ点・ばらつき）
- [ ] MSA（Ga R&R・バイアス・直線性）で測定の信頼性を確認★
- [ ] 測定環境（温湿度・振動）の管理
- [ ] 不適合測定器の処置と**遡及調査**（その器で測った製品の扱い）★
- [ ] 記録の保管

## D3. 監査対応（内部監査・顧客監査・審査）
**考え方**：監査は減点回避でなく“改善機会”。

- [ ] 内部監査：年間計画・チェックリスト・是正フォロー
- [ ] 顧客監査：事前準備（文書・記録・現場5S）・想定問答
- [ ] 審査（ISO 9001／14001／13485）の維持・更新管理★
- [ ] 指摘事項の是正と**水平展開**（同種の他箇所も直す）★
- [ ] 文書・記録の整合（言っていることと記録が一致）
- [ ] 是正の期限管理とエビデンス
- [ ] 監査結果をマネジメントレビューへ

> 【要確認】保有認証の更新時期・内部監査の頻度。

## D4. 化学物質管理・法規対応
- [ ] 適用される法規制を把握（RoHS・REACH・各国規制）★
- [ ] 顧客ごとの要求基準を把握
- [ ] 成分情報の収集（SDS・IMDS・chemSHERPA）
- [ ] 含有調査・非含有保証書の管理★
- [ ] 材料・工程変更時に再調査（変更管理と連動）★
- [ ] 法改正・規制追加のウォッチ
- [ ] 記録・トレーサビリティ

## D5. 文書・記録管理
- [ ] 文書の版管理（最新版のみ現場に・旧版回収）★
- [ ] 承認・配布・改廃のルール
- [ ] 記録の保管期間・保管場所・アクセス権限★
