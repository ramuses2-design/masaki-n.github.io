# 場面別プレイブック集（網羅版）

> **位置づけ**：方針マスター（考え方・ルール）を「いつ・どの場面で・何をするか」に落とした**行動指針**。
> 各場面の手順とチェック項目に沿えば、やり方に漏れが出ない状態を目指す。
> **凡例**：`【要確認】`＝あなた（GL）の確定が必要な箇所。`★`＝特に外すと事故につながる重点項目。
> **共通の型**：① 事実確認 → ② 影響評価 → ③ 打ち手 → ④ 記録 → ⑤ 横展開（どの場面でも同じ順番で回す）。

## 場面マップ（5グループ）
- **A 定例・マネジメント**：A1 1on1／A2 週次／A3 月次／A4 目標設定・評価／A5 教育・力量管理・新任受け入れ
- **B 異常対応**：B1 トリアージ／B2 初動・封じ込め／B3 原因究明／B4 是正・恒久・横展開／B5 特採申請／B6 顧客連絡・対外／B7 重大品質問題・リコール級
- **C 変化・プロジェクト**：C1 4M変更管理／C2 新製品・量産立ち上げ／C3 設備導入・改善／C4 工程・検査基準の制定改訂
- **D 予防・維持・システム**：D1 日常検査・検査表運用／D2 測定器・校正・MSA／D3 監査対応／D4 化学物質・法規／D5 文書・記録管理／D6 KPI・品質目標
- **E 教育・OJT**：E1 OJT計画の立て方／E2 仕事の教え方の基本

# A. 定例・マネジメント

## A1. 1on1（育成・面談）
**目的**：育成・負荷把握・状態確認の双方向の場。進捗の詰問・報告会にしない。

**準備（GL）**
- [ ] 前回のアクション・約束を確認しておく
- [ ] 本人の最近の実績・負荷データ（工数・不良対応）に目を通す
- [ ] 今日のテーマを1つ用意（毎回“育成の問い”を持つ）

**進め方（傾聴7割）**
- [ ] ①コンディション（体調・負荷・困りごと）をまず聞く
- [ ] ②業務の障害（自分では外せない壁）を引き出す → GLが外す
- [ ] ③成長（スキルマップの現在地と次の一歩・任せたい役割）
- [ ] ④改善テーマの進み（改善の時間が取れているか）
- [ ] ⑤キャリア・要望（中期で挑戦したいこと）
- [ ] 最後に「次アクションと期限」を一緒に決める

**GLが必ず確認（観察）★**
- [ ] 過負荷・疲弊の兆候（残業・表情・口数）
- [ ] 属人化して休めない業務はないか（副担当の要否）
- [ ] 改善・学習の時間が取れているか
- [ ] 安全・コンプライアンス・ハラスメントの懸念
- [ ] モチベーション低下のサイン

**やらないこと（NG）**
- 進捗報告だけで終える／一方的に指示して終える／不満を放置する／評価面談と混同する

> 【要確認】頻度（例：月1回30分）・記録の残し方。

**1on1で使える質問例（テーマ別・全30問）**

①コンディション
1. 最近、体調や睡眠で気になることはある？
2. ここ2週間で一番しんどかった場面は？
3. 残業が続いているけど、無理はしていない？
4. 今の仕事量は自分の感覚としてどう？
5. 休みは取れている？取りづらい理由はある？
6. 何かひとりで抱えていることはない？

②業務の障害
7. 今、自分では外せない壁になっていることは？
8. 他部門・他工程との連携で困っていることは？
9. 情報や指示が足りずに困った場面は最近あった？
10. GLに動いてほしいことはある？
11. 道具・設備・システムで不便に感じているところは？
12. もし何でも変えられるとしたら、何を変えたい？

③成長
13. 今、スキルマップ上で伸ばしたいところはどこ？
14. 最近「できるようになった」と感じたことは？
15. 次に任せてみたい役割・作業はある？
16. 教える側に回ってみたいと思うことはある？
17. 苦手意識がある作業・場面はある？
18. 学びたい・挑戦したい研修や知識はある？

④改善テーマ
19. 今取り組んでいる改善は順調に進んでいる？
20. 改善のための時間は実際に取れている？
21. 改善を進める上で協力してほしいことはある？
22. 小さいけど気になっている「ムダ」「やりにくさ」はある？

⑤キャリア・要望
23. 半年〜1年後、どんな役割を担っていたい？
24. 今の働き方で続けにくいと感じる部分はある？
25. 異動・別工程への挑戦に関心はある？
26. 会社・グループに要望したいことはある？
27. プライベートとの両立で気になることはある？

締めくくり
28. 今日話した中で一番大事だと思ったことは？
29. 次回までにやってみたいことは？
30. 言い残したことはない？

**やってはいけない質問・言い方（NG例）**
- 「なんで進捗遅いの？」など詰問から入る（1on1を報告会・詰問の場にしない）★
- 「他の人はできているけど」と比較で追い込む
- 沈黙をすぐに埋めてしまい、本人の言葉を待たない
- 個人の性格・私生活を評価するような聞き方をする（決めつけで話す）
- 聞いたことをその場で否定・説教する（まず最後まで聞く）★
- 全質問を機械的に読み上げる（対話ではなく尋問にしない）

関連場面: A4, A5

## A2. 週次ミーティング（グループ内）
**目的**：異常の共有と、月次に出す数字・メッセージを先に揃える（伝達のブレ防止）。

**アジェンダ**
- [ ] 異常・不適合・ヒヤリの共有（先週発生分）★
- [ ] 進捗（計画対実績）と未達の要因
- [ ] 不良傾向（パレート・推移）
- [ ] 負荷状況（偏り・属人化のサイン）
- [ ] 改善テーマの進捗（5〜10%が機能しているか）
- [ ] 来週の重点と役割分担
- [ ] 月次に出す数字・結論の確定★

**確認チェック**
- [ ] 異常・不適合の上げ漏れはないか
- [ ] 即エスカレーションすべき案件はないか
- [ ] 負荷の偏り・属人化の再配分は要るか
- [ ] 先週のアクションはクローズしたか
- [ ] 月次の数字を全員が同じ言葉で言えるか

関連場面: A3, A8, D6

## A3. 月次報告（部門横断：製造係長＋他GL＋事業部長）
**目的**：意思決定に必要なことを、結論ファーストで上げる。

**報告の型（順番）**
1. 結論（今月、品質はどうか・何を決めてほしいか）
2. 数字（KPI：不良率・クレーム/不適合件数・是正の状況・改善進捗）
3. 重要案件（顧客・監査・重大不適合の状況と対応）
4. 改善ロードマップの進捗（アプリ移行・検査表標準化 等）
5. 依頼・エスカレーション（誰が・いつまでに）

**準備チェック**
- [ ] 数字は週次でグループ内と一致しているか★
- [ ] 悪い情報を先に出す構成か（後回しにしない）
- [ ] 依頼は「誰が・いつまでに」まで言えるか
- [ ] 前月の依頼・宿題のクローズ状況を示せるか

> 【要確認】月次KPIの正式定義・目標値・フォーマット。

**月次報告フォーマット例（記入例・架空）**
| 項目 | 記入例 |
|---|---|
| 結論 | 今月の品質は概ね安定。ただしX工程で不良率が前月比+0.3pt上昇しており対策中。監査への準備状況について判断を仰ぎたい |
| 数字 | 不良率1.2%（前月1.0%）／クレーム件数2件（前月1件）／是正処置 進行中2件・完了3件／改善進捗 目標比80% |
| 重要案件 | 特採対応1件（原因調査中、来週回答予定）／来月に認証更新監査あり |
| 改善ロードマップ | 検査表の標準化：計画比90%進捗。次月に全工程展開完了予定 |
| 依頼・エスカレーション | 設備更新の予算判断を〇月〇日までにお願いしたい |

**数字の見せ方（前月比・傾向・特異点）**
- [ ] 単月の数字だけでなく前月比・直近3か月推移をセットで示す（一時的な悪化か傾向かが分かるように）★
- [ ] 良い数字も悪い数字も同じ粒度で見せる（悪い情報だけ丸めない）
- [ ] 特異点（急上昇・急低下）があれば、原因の見当（分かっている範囲）を一言添える★
- [ ] グラフ・表は「結論を支える最小限」にとどめ、詳細は別紙・添付に回す
- [ ] 目標値・基準値を必ず併記し、達成／未達が一目で分かるようにする

関連場面: A2, D6

## A4. 目標設定・評価（期初／期中／期末）
**目的**：組織目標を個人に接続し、事実に基づいて公正に評価・育成する。

**期初（目標設定）**
- [ ] 事業部・グループ目標を個人目標へブレイクダウン
- [ ] SMART（具体・測定可能・達成可能・関連・期限）で設定
- [ ] 「業務目標」＋「スキル目標」＋「改善目標」をセットで
- [ ] 評価軸・期待役割を本人と共有（後出しにしない）★

**期中（モニタ）**
- [ ] 1on1で進捗を確認、つまずきを早期に支援
- [ ] 環境変化があれば目標を見直す（固定しない）

**期末（評価）**
- [ ] 事実・データで評価する（印象で決めない）★
- [ ] 本人の自己評価とすり合わせる
- [ ] フィードバックは行動ベースで具体的に
- [ ] 良い点と次の課題を両方伝え、次期へつなぐ
- [ ] 評価基準は全員に一貫しているか（えこひいきの排除）

> 【要確認】評価軸の中身・評価サイクル（半期/通期）・処遇との連動。

関連場面: A1, A5, D6

## A5. 教育・力量管理／新任受け入れ
**目的**：必要な力量を定義し、計画的に育て、記録する（ISO の力量管理）。

**スキルマップ運用**
- [ ] 業務ごとに必要力量レベルを定義
- [ ] 各人の現在地を評価しギャップを可視化
- [ ] ギャップに教育・OJT計画を割り当て
- [ ] 力量の認定・記録を残す★
- [ ] 重要工程は認定者のみが担当（無資格者にやらせない）★

**新任受け入れ（オンボーディング）**
- [ ] ①安全教育 → ②品質の基本・ルール → ③OJT（副担当を付与）
- [ ] 独り立ち判定の基準を事前に決める
- [ ] 一定期間後にフォロー面談（定着確認）
- [ ] 教育記録を残す

**外部研修・セミナーを使う場合**
- [ ] 社内でOJTしきれない専門領域（統計手法・信頼性・法規）は、外部研修を計画に組み込む★
- [ ] 教育訓練の費用は公的助成の対象になり得るが、多くは訓練開始前の計画届出が必要。研修計画の段階で対象可否を確認する（受講後では間に合わないことがある）★
- [ ] 受講後は、内容をグループへ共有し力量認定・教育記録に紐づける（受けっぱなしにしない）

> 【要確認】スキルマップの項目・力量認定の基準・新任の標準育成期間、外部研修の予算枠と申請フロー。

関連場面: A1, A11, M8, M9, C7

## A6. 改善提案・小集団活動（QCサークル）の運営
**目的**：現場発の改善を止めずに回す。テーマ選定から効果確認・標準化までを型にする。

**運営の型**
- [ ] 職場の困りごと・不良/ロスからテーマを選ぶ（データで優先順位）★
- [ ] 現状把握 → 要因解析 → 対策 → 効果確認（QCストーリー）で進める
- [ ] 対策は標準（作業標準・帳票）へ反映して定着させる★
- [ ] 成果を発表・共有し、他職場へ横展開する

**続ける工夫**
- [ ] 小さく始め、短サイクルで回す（完璧を狙わない）
- [ ] 参加者の負荷に配慮し、時間を確保する
- [ ] 【要確認】活動の頻度・テーマ登録/表彰の仕組み・工数の扱い

関連場面: A2, D6, C1

## A7. 協力会社・外注先との定例品質会議の運営
**目的**：外注・協力先とのトラブルを未然に防ぐ。定例で品質情報・変化点・改善を共有し、関係を「発注—受注」から「共同改善」へ育てる。

**運営の型**
- [ ] 不適合・クレーム・4M変更・納入品質の推移を定例アジェンダに固定する★
- [ ] 相手先の工程変更・製造中止の予告を早期に吸い上げる（C5と連動）★
- [ ] 改善テーマを一緒に決め、次回までの宿題と担当を明確にする
- [ ] 決定事項・依頼事項は議事録で残し、社内の購買・品証へ展開する

**続ける工夫**
- [ ] 相手を責める場にせず、事実とデータで建設的に話す
- [ ] 重要先から始め、頻度は負荷に応じて調整する
- [ ] 【要確認】対象取引先・開催頻度・議事録様式・エスカレーション基準

関連場面: C5, P6, B4

## A8. 品質KPIの朝礼・見える化ボード運用（現場への浸透）
**目的**：品質目標を「管理者だけが見る数字」で終わらせない。朝礼と見える化ボードで日々の品質状況を共有し、現場全員で異常に気づける状態をつくる。

**運営の型**
- [ ] 不良率・重要クレーム・当日の注意ロット等、現場が見て動ける指標を厳選する★
- [ ] 朝礼で前日実績と今日の注意点を短時間で共有する（数字＋一言）★
- [ ] 見える化ボード（現物/デジタル）を更新し、目標と実績の差を可視化する
- [ ] 異常・逸脱はその場で当日の対応につなげ、翌日フォローする

**続ける工夫**
- [ ] 数字を「責める道具」にせず、改善のきっかけとして扱う
- [ ] 更新はkintone/SharePoint等で自動化し、集計の手間を減らす
- [ ] 【要確認】共有する指標・ボードの様式・更新頻度・朝礼の運用

関連場面: A2, D6, M5, A9

## A9. 品質月間・全社品質イベントの企画運営（年間の品質意識向上）
**目的**：品質意識を単発でなく年間の活動として根付かせる。品質月間（11月）などの節目を使い、現場を巻き込んだ啓発・教育・改善発表の場をつくる。

**企画の型**
- [ ] 年間計画に品質月間・発表会・教育を位置づけ、テーマと目標を決める★
- [ ] 現場が主役になる企画（QCサークル発表・標語・改善事例共有）を組む★
- [ ] 外部資源（品質月間テキスト・セミナー・展示会）を活用し内容を補強する
- [ ] 経営層・他部門を巻き込み、成果を全社で共有する

**続ける工夫**
- [ ] やりっぱなしにせず、優れた改善は標準・横展開（B4）につなげる
- [ ] 参加状況・改善件数などを記録し、翌年の企画に反映する
- [ ] 【要確認】年間の品質イベント計画・予算・事務局体制

関連場面: A6, A8, D6

## A10. 顧客監査・第三者審査（ISO等）の受審準備と当日運営
**目的**：顧客監査やISO等の審査を、指摘を恐れる場ではなく「体制を点検し改善につなげる機会」として計画的に運営する。

- [ ] 監査/審査の範囲・基準・過去指摘を事前に確認し、担当と役割を割り当てる★
- [ ] 想定質問・必要記録（手順・力量・是正・トレーサビリティ）を事前に整える★
- [ ] 現場（5S・掲示・現物・測定器）を日常の延長で整える（付け焼き刃にしない）
- [ ] 当日は事実ベースで回答し、分からない点は持ち帰る（推測で答えない）
- [ ] 指摘・観察事項は是正処置(B4)として登録し、期限・担当・有効性確認まで管理する
- [ ] 【要確認】対応窓口・回答権限、記録の開示範囲、是正の完了判定者

関連場面: S6, D3, B4

## A11. 展示会・外部セミナーの視察（狙いを決めて行き、持ち帰りを形にする）
**目的**：展示会・セミナーを「行っただけ」で終わらせない。事前に検討テーマを決めて行き、持ち帰りを次のアクション（検討・比較・教育）に接続する。

**行く前に決める**
- [ ] 検討中のテーマを1〜2件に絞って持って行く（漠然と回ると情報量に埋もれる）★
- [ ] 現状の困りごとを数字で言えるようにしておく（工数・不良率・リードタイム）。出展者との会話の密度が変わる★
- [ ] 出展者リスト・セミナー枠を事前に見て、優先順位と回る順を決める
- [ ] 【要確認】出張申請・費用負担・報告書の様式と提出先

**当日の型**
- [ ] 同種のソリューションは必ず複数社を回り、比較できる状態にする（1社だけ見ると判断を誤る）★
- [ ] 「自社の条件（材料・速度・環境）で成立するか」を具体的に聞く。カタログ値をそのまま信じない
- [ ] その場で契約・約束をしない。持ち帰って社内で評価する

**持ち帰りを形にする**
- [ ] 帰社後すぐ（記憶が薄れる前）に要点をメモ化し、グループへ共有する★
- [ ] 検討に進むものは、設備なら C3-a／自動検査なら T10／ノーコード自動化なら T15 へ接続する
- [ ] 教育目的のセミナーは、受講内容を力量管理・教育記録（A5）に紐づける
- [ ] 見送るものも「なぜ今回は見送るか」を残す（同じ検討の繰り返しを防ぐ）

関連場面: A5, A9, C3-a, T10, T15

## A12. 部門間コンフリクトの調整（品質と納期・コストの対立場面）
**目的／考え方**：品質と納期・コストが対立する場面は避けられない。原則は「安全・法規に関わる品質基準は交渉しない、それ以外は影響とリスクを数字で並べて判断する」。感情論・声の大きさで押し切らない。

- [ ] 対立の論点を「事実」と「意見・立場」に分けて整理する（何が譲れないラインかを最初に確認）★
- [ ] 安全・法規・顧客契約に関わる品質基準は交渉材料にしない（このラインは明確に伝える）★
- [ ] 納期・コスト側の要求は、遅延・追加コストの影響を具体的な数字で出してもらう
- [ ] 品質側も「基準を緩めた場合のリスク（流出確率・想定損害）」を数字や過去事例で示す★
- [ ] 両部門だけで平行線のときは、早めに上位者（GL・部門長）を巻き込み、判断を持ち越さない
- [ ] 決定事項と理由は記録に残し、関係者へ共有する（同じ対立の蒸し返しを防ぐ）
- [ ] 決定後は「なぜその判断になったか」を現場にも分かる言葉で説明する
- [ ] 【要確認】最終判断権限者・エスカレーション先の社内基準

関連場面: B5, S5, D6

## A13. 品質保証の年間計画・予算要求の立て方
**目的／考え方**：年間計画・予算は思いつきで積み上げると承認されにくい。前年実績・重点リスク・投資対効果を根拠として整理し、経営判断に乗せやすい形にする。

- [ ] 前年度の実績（クレーム件数・不良率・監査指摘等のKPI推移）を整理し、計画の根拠にする★
- [ ] 今年度の重点リスク（設備老朽化・法規改正・新規顧客要求等）を洗い出し、優先順位をつける★
- [ ] 教育・認定（E章）・設備更新・システム投資など、予算が必要な項目を洗い出す
- [ ] 各項目の投資対効果（防げる損失・工数削減）を可能な範囲で数値化する★
- [ ] 予算要求書は「現状の課題→対策→必要金額→期待効果」の順で1枚に整理する
- [ ] 承認者への説明前に、関連部門（技術・製造・購買）と事前にすり合わせる
- [ ] 未確定・交渉中の項目は金額を仮置きにせず、幅（レンジ）で示す
- [ ] 【要確認】予算要求の様式・提出時期、決裁権限者の社内基準

関連場面: A3, D6, D10

# B. 異常対応（不適合・クレーム）

> 合言葉＝**「まず止める、それから直す」**。スピードと優先順位がすべて。一人で抱えない。

## B1. 優先順位（トリアージ：上から確認）
**目的／考え方**：異常の第一報を受けたら、原因を考える前に「どこまで急ぐか」を決める。順番を上から確認するのは、下位の損得を先に考えると上位の判断が遅れるため。**迷ったら上位（安全側）に倒す**——後から下げるのは容易だが、上げ直すときには既に流出している。

1. **人の安全**：人身・労災・安全に関わるか ★
2. **法規制・コンプラ**：法令・規制（化学物質・安全基準等）違反の恐れ ★
3. **顧客への影響**：ライン停止・出荷済み・市場流出の恐れ ★
4. **流出範囲**：社内でどこまで波及（ロット・数量・期間）
5. **社内損失**：手直し・廃棄・納期影響

**トリアージの型（第一報から30分以内に決める）**
- [ ] 1〜5のどこに当たるかを、上から順に一つずつ確認する（下から見ない）★
- [ ] 「出荷済みか／社内で止まっているか」を最初に確認する（この一点で緊急度が変わる）★
- [ ] 判断がつかない項目は「該当する」側に倒して先へ進む（確認待ちで止めない）★
- [ ] 決めた優先度と、その根拠になった事実（ロット・数量・出荷状況）を記録に残す
- [ ] 優先度は固定しない。事実が増えたら上げ下げし、変えたことを関係者へ伝える

| 該当レベル | 連絡のしかた | 初動の目安 |
|---|---|---|
| 1〜3（安全・法規・顧客影響） | **即・GLへ直接**（メール／Teams／電話／口頭）。返信を待たず並行して着手 ★ | 出荷停止・在庫隔離を先にかける（B2）。重大なら B7 へ |
| 4（社内での波及） | その日のうちにGLへ報告し、対象ロットの範囲を共有 | 前後ロット・同条件品まで広げて確認（B2） |
| 5（社内損失のみ） | 定例（週次）で報告し、傾向として扱う | 通常の是正フロー（B3〜B4）で処理 |

> 1〜3 のいずれかに当たれば **即・GLへ直接**（メール／Teams／電話／口頭）。連絡が遅れる理由の大半は「まだ調べ切れていないから」だが、**事実が固まる前の第一報が正しい**。分かっている範囲だけで伝える。

**やってはいけない（NG）**
- 原因が分かってから報告しようとして、出荷・流出が進む★
- 「たぶん大丈夫」で1〜3の該当を自分だけで否定する

関連場面: B2, B7, M2, B6

## B2. 初動・封じ込め（流出を止める）
- [ ] **現品・現物の確保**（不良品・現物・ラベル・ロットを動かす前に記録／写真）★
- [ ] **封じ込め**：出荷停止・在庫の隔離・該当品の識別（赤札等）・仕掛品の確認★
- [ ] **影響範囲の特定**：ロットトレース（前後ロット・同条件品・他顧客・他品種への波及）
- [ ] **数量・期間の特定**（いつから・どれだけ）
- [ ] **暫定（応急）対策**：全数検査・選別・代替品手配で安全側へ
- [ ] **顧客への一次連絡**：事実・暫定対応・調査予定・回答期限（対外はGL／窓口）★
- [ ] **社内連携**：製造・営業・関係部門へ即共有

**初動1時間チェックリスト（時系列）**
1. 0〜10分：第一報を受けたらB1トリアージを実施し、レベルを仮決定する ★
2. 10〜20分：現品・現物を確保し、写真・ラベル・ロット番号を記録する（動かす前に記録）★
3. 20〜30分：出荷停止・在庫隔離など暫定の封じ込めを実施する
4. 30〜45分：影響範囲（前後ロット・同条件品・他顧客）の見当をつける
5. 45〜60分：社内一報を送り、関係部門（製造・営業・品証）へ共有する ★

**社内一報のテンプレ文例（第一報に入れる5項目・架空例）**
> 件名：【初動】〇〇工程 不適合発生（レベル2・調査中）
> ①何が：〇〇部品にてサイズ不良を確認
> ②いつ・どこで：本日10時、〇〇工程の検査で発覚
> ③現在の対応：出荷停止・在庫隔離を実施済み
> ④影響範囲（現時点）：対象ロット〇〇、前後ロットは調査中
> ⑤次の連絡予定：15時までに続報

> 第一報は「事実が固まってから」ではなく、分かっている範囲だけで**60分以内**に出す。空欄があってもよい。

関連場面: B1, B3, M2, S3

## B5. 特採（特別採用）申請の判断
**考え方**：規格外＝即NGではない。ただし安全・法規は絶対に妥協しない。常態化させない。

- [ ] **安全・法規に関わるものは特採しない**（絶対）★
- [ ] お客様の用途・機能に影響しないかを確認
- [ ] **顧客の承認が必須**（無断で出荷可否を変えない）★
- [ ] 根拠データを添付（規格・試験・実績）
- [ ] 適用範囲・数量・期限を限定（恒久措置にしない）
- [ ] 出荷可否の最終判断はGL
- [ ] 恒久対策（規格外を出さない仕組み）とセットで進める
- [ ] 申請・承認・適用を記録

**特採判断の流れ（テキストフロー）**
1. 規格外が発生 → まず「安全・法規に関わるか」を確認 → 関わる場合は特採不可、恒久対策へ ★
2. 安全・法規に関わらない場合 → 顧客の用途・機能への影響を技術と確認
3. 影響が許容範囲と判断できる場合 → 申請書を作成し、根拠データを添付
4. GLが最終判断 → 必要に応じて顧客承認を取得（顧客要求がある場合は必須）★
5. 適用範囲・数量・期限を限定して適用 → 記録を残す
6. 恒久対策（同じ規格外を出さない仕組み）を並行して進める

**特採申請書に入れる項目（記入例・架空）**
| 項目 | 記入例 |
|---|---|
| 対象ロット・数量 | ロットNo.〇〇、〇〇個 |
| 逸脱内容 | 規格〇〇mmに対し実測〇〇mm |
| 影響評価 | 顧客用途に対する機能影響なし（技術部門確認済） |
| 根拠データ | 試験成績書No.〇〇、過去実績〇件 |
| 適用範囲・期限 | 当該ロットのみ・〇月〇日出荷分限り |
| 承認 | 申請：〇〇／承認：GL／顧客承認：要・不要 |

> 【要確認】特採申請書の正式様式・保存先、顧客承認が必要となる条件の線引き。

関連場面: S4, B6

## B6. 顧客連絡・対外対応の作法
- [ ] 窓口はGL／対外担当（現場が直接抱え込まない）★
- [ ] 一次連絡は早く（事実・暫定対応・調査予定・回答期限）
- [ ] 憶測で語らない／確定していないことを約束しない★
- [ ] 約束は守れる範囲で、守れる期限で
- [ ] 是正回答の期限を管理（遅れそうなら事前連絡）
- [ ] 報告書の品質を担保（事実・原因・対策・再発防止）
- [ ] 報告後の関係維持・信頼回復まで見る

**顧客への一報メール文例（架空例・良い例／NG例の対比）**

良い例：
> 件名：〇〇の件のご連絡（第一報）
> いつもお世話になっております。〇〇の件につき、本日〇時に社内で不適合を確認いたしましたのでご連絡いたします。現時点で判明している内容は以下の通りです。①事実：〇〇　②当社の暫定対応：出荷停止・在庫確認済み　③調査予定：原因究明中、〇日までに中間報告いたします。ご不便をおかけし申し訳ございません。

NG例：
> 件名：ご報告
> 大変申し訳ございません。至急確認しまして、原因が分かり次第また改めてご連絡いたします。

**良い例／NG例の対比**
- [ ] 事実（何が・いつ）を最初に書く／NG：謝罪だけで始めて事実が後回しになる
- [ ] 暫定対応（止めた・隔離した）を明記する／NG：「対応中です」とだけ書き、何をしたか分からない
- [ ] 次回連絡の期限を約束する／NG：「分かり次第」とだけ書き、いつまでか分からない ★
- [ ] 分かっている範囲だけで送る／NG：原因が分かるまで送らない（B1の原則と矛盾）

> 【要確認】顧客テンプレの正式文面・宛先ルール（窓口一本化の範囲）。

関連場面: S3, B7, B8

## B10. 是正処置の有効性確認（効果検証）とクローズ・再発監視
**目的**：対策を「打った」で終わらせない。一定期間の実績で効果を確認し、再発がないことを見届けてから正式にクローズする。

**検証の型**
- [ ] 対策後の実績（不良率・再発件数）を基準・目標と比較する期間を決める★
- [ ] 効果が不十分なら原因究明（B3）に戻し、対策を見直す★
- [ ] 有効性が確認できたら標準・帳票・教育への反映を確認する（B4）
- [ ] 顧客提出済みの是正報告（B8）があれば、クローズ状況を整合させる

**クローズと監視**
- [ ] クローズ基準（期間・件数・確認者）を満たしたことを記録して正式終了
- [ ] 横展開先でも再発がないか、一定期間モニタリングする
- [ ] 【要確認】有効性確認の期間・判定基準・クローズ権限者

## B11. 難航・長期化案件の是正処置マネジメント（暫定対策の維持と進捗管理）
**目的**：真因究明や恒久対策に時間がかかる案件で、暫定対策（歯止め）を確実に効かせ続けながら、案件を放置・空中分解させずにクローズへ導く。

- [ ] 暫定対策（全数検査・隔離・工程条件の制限等）が継続して機能しているか定期的に確認する★
- [ ] 恒久対策の進捗・障害を見える化し、担当・期限を明確にして定例でレビューする★
- [ ] 長期化の理由（技術的難易度・費用・他部門待ち）を整理し、必要な支援・決裁を上位に上げる
- [ ] 暫定対策のまま常態化させない（惰性で全数検査を続けない）よう期限を設ける
- [ ] 恒久対策の有効性を確認(B10)してからクローズし、暫定対策の解除条件を記録する
- [ ] 【要確認】暫定対策の解除判定者・レビュー頻度、長期案件のエスカレーション基準

## B12. 記録不備・データの信頼性に疑義があるときの対応（発見時の初動）
**目的**：検査記録や工程データの改ざん・記入漏れ・つじつまの合わなさに気づいたときは、個人を責める前に事実関係を止めて確認する。放置すると記録全体の信頼性が崩れる。

- [ ] 疑義のある記録・データを特定し、原本（電子データ含む）を保全する（上書き・破棄させない）★
- [ ] 対象ロット・期間の出荷可否をいったん保留し、品証GLへ速やかに報告する★
- [ ] 記入者個人への詰問より先に、事実確認（いつ・何が・どの範囲で）を淡々と行う★
- [ ] 影響範囲（対象ロット・出荷先・他の記録への波及）を洗い出す★
- [ ] 是正（再検査・出荷停止・顧客連絡の要否）をB1の優先順位に沿って判断する
- [ ] 恒久対策は「個人の意識」だけに頼らず、記録の仕組み（ダブルチェック・システム化）で防ぐ方向で検討する
- [ ] 【要確認】記録不備発覚時の報告フロー、懲戒に関する社内規程との関係

関連場面: B1, D5, M16

## B13. 官公庁・認証機関への報告が必要な事案の対応
**目的／考え方**：不具合の中には、社内対応だけでなく法令・認証制度上、官公庁や認証機関への報告が必要になるものがある。顧客対応で手一杯になり報告を忘れる・遅れることを防ぐため、該当基準と窓口を事前に整理しておく。

- [ ] 対象となりうる事案（重大な安全性不具合、認証（ISO等）に関わる不適合、業法上の報告義務がある事象）を洗い出す★
- [ ] 事案発生時、報告要否の一次判断をGL・品証で行う（現場の自己判断で「報告不要」と決めない）★
- [ ] 報告が必要な場合、期限・様式・提出先（監督官庁／認証機関）を確認する★
- [ ] 社内の事実関係（発生日・原因・対象範囲・是正状況）を、報告書に必要な粒度で整理する★
- [ ] 認証機関への報告は、認証の維持・更新審査への影響も踏まえて対応する
- [ ] 報告後のフォローアップ（追加質問・是正状況の報告）の窓口を一本化する
- [ ] 報告の記録（提出日・内容・やり取り）を保管し、次回同種事案の参照にする
- [ ] 【要確認】報告義務の対象となる法令・認証制度の一覧、報告様式・提出窓口

関連場面: B7, D3, B6

## C1. 4M変更管理（人・機械・材料・方法）
**最大のリスクは“暗黙の変更”**。気づかぬ変更がクレームの主因になる。

- [ ] 変更の検知（材料・設備・工法・人・測定・環境のどれか）★
- [ ] 変更前に影響評価とリスクアセスメント
- [ ] **顧客承認の要否を確認**（契約・仕様・4M変更届）★
- [ ] 妥当性確認（初物・工程能力・試験）
- [ ] 変更履歴を記録（いつ・何を・なぜ・誰が）
- [ ] 関係者へ周知・教育
- [ ] 変更後フォロー（初期流動で異常がないか）
- [ ] **無断変更の禁止を全員に徹底**★

> 【要確認】顧客承認が必要になる変更の社内線引き。

**4M変更届の記入例（架空・4種）**
| 区分 | 記入例 |
|---|---|
| 人 | 変更内容：主要工程の担当者交代（新任者、教育未了）／理由：異動／影響評価：新任者はA5の力量マップで未認定のため教育完了まで先輩者立会い／顧客承認：不要（社内変更のみ、契約上の承認対象外） |
| 機械 | 変更内容：成形機の予備機への切替（同型式・別号機）／理由：主機の定期整備／影響評価：初物確認・工程能力（Cp/Cpk）を再確認／顧客承認：不要（同型式・同条件のため） |
| 材料 | 変更内容：原料メーカーの製造ロット切替（規格同等品）／理由：定期切替／影響評価：受入検査＋初期数ロットの特性確認／顧客承認：契約上「材料メーカー変更は要承認」のため申請中 |
| 方法 | 変更内容：検査工程の一部を目視からセンサー検査へ変更／理由：省力化・検出力向上／影響評価：判定基準の相関確認（新旧比較データ取得）／顧客承認：【要確認】検査方法の変更が承認対象かどうか |

**記入時のNG例**
- [ ] 「軽微な変更なので届出は不要」と自己判断する（軽微かどうかは品証が判断する）★
- [ ] 変更理由を書かず結果だけ記載する（影響評価の妥当性が判断できなくなる）
- [ ] 顧客承認の要否欄を空欄のまま提出する（要否不明のまま流さない）★

## C7. 公的支援（補助金・助成金）を使う設備・システム投資の進め方
**目的**：補助金・助成金を前提にした投資を、制度に振り回されずに進める。制度に合わせて計画を歪めず、あくまで自社の課題起点で組み立てる。

**前提（つまずきやすい点）**
- [ ] 制度は「締切・要件が頻繁に変わる」前提で、必ず公式の公募要領を一次情報として確認する（解説記事だけで判断しない）★
- [ ] 申請の入口（電子申請アカウント等）は取得に時間がかかることがある。検討段階で先に着手しておく★
- [ ] 多くの制度は「事前申請・交付決定後に着手」が原則。先に発注・契約すると対象外になり得る★
- [ ] 教育訓練系の助成は、訓練開始前の計画届出が必要な場合がある（受講後の申請では間に合わない）

**組み立ての型**
- [ ] 先に自社の課題と投資目的を固める。制度に合わせて目的を後付けしない（採択されても使われない設備になる）★
- [ ] 制度が求める目的（省力化・デジタル化・省エネ 等）と自社の目的が、無理なく重なる制度を選ぶ
- [ ] 補助金は原則あと払い。採択されても一旦は自己負担が発生する前提で資金計画を見る
- [ ] 対象可否が微妙なものは、思い込みで判断せず事務局へ照会する
- [ ] 【要確認】社内の投資決裁フロー・申請の主管部署・資金計画の相談先

**採択後・不採択時**
- [ ] 採択後は実績報告・証憑保管・効果報告の義務が続く。誰が担当するかを申請前に決めておく★
- [ ] 不採択でも計画は資産。次回公募や通常の設備投資計画へ回す
- [ ] 導入後は当初の目的（工数・不良率等）が実際に改善したかを検証する（C3-b）

関連場面: C3-a, C3-b, A5, T15

## C8. 生産移管（ライン移設・工場間・外注化）の品質保証
**目的**：生産移管は「同じ物を作っているつもり」が最大の落とし穴。設備・環境・人が変われば工程は別物として扱い、移管前後で品質を作り込み直す。

- [ ] 移管理由・スコープ（全数移管か一部か、時期）を最初に明確化する★
- [ ] 移管元と移管先の設備・治具・測定器の差分を洗い出す（同型番でも個体差・据付環境差はあるものとして扱う）★
- [ ] 材料・副資材の入手性・仕様差（サプライヤー変更を伴う場合は特に）を確認する
- [ ] 作業標準・検査基準・限度見本を移管先向けに整備し直す（「そのまま持っていけば大丈夫」と思い込まない）★
- [ ] 移管先の要員教育・力量評価を実施し、記録を残す（E章と連携）
- [ ] 初期流動管理（移管後の重点監視期間・監視項目）を設定する★
- [ ] 顧客への事前連絡・承認要否を確認する（4M変更届の対象になり得る）★
- [ ] 移管完了の判定基準（工程能力・初物確認・関係者の合意）を事前に決めておく
- [ ] 【要確認】移管計画書の様式・承認フロー・顧客通知の要否判断基準

関連場面: C1, C2, E1

## C9. 急な増産・特急対応時の品質リスク管理
**目的**：特急対応は「いつも通り」ではない。人・設備・時間の無理がどこにしわ寄せされるかを事前に見極め、品質を落とさず対応する。

- [ ] 増産・特急対応で変わる条件（人員・シフト・設備稼働率・材料調達）を洗い出す★
- [ ] 応援者・多能工の投入がある場合は、最低限の教育・力量確認を省略しない（M9・E7と連携）★
- [ ] 検査・記録の工程を安易に省略しない。省略する場合は根拠と承認者を明確にする★
- [ ] 材料・副資材の緊急調達がある場合、通常ルートと同等の受入基準を維持する★
- [ ] 増産期間中の異常発生率・不良率を通常時と比較してモニタする
- [ ] 対応終了後、無理な運用が常態化していないか振り返る
- [ ] 【要確認】特急対応時に検査・記録を簡略化してよい範囲、承認者

関連場面: M9, E7, P7

## D2. 測定器・校正・MSA管理
**目的**：「測れていない」状態は不良の見逃しと同義。校正切れ・測定ばらつきを仕組みで防ぎ、測定の信頼性を担保する。

- [ ] 校正計画と有効期限の管理（期限切れ使用ゼロ）★
- [ ] 校正ラベル・識別
- [ ] 社内校正／外部校正の区分と基準器のトレーサビリティ
- [ ] 使用前点検（ゼロ点・ばらつき）
- [ ] MSA（Gage R&R・バイアス・直線性）で測定の信頼性を確認★
- [ ] 測定環境（温湿度・振動）の管理
- [ ] 不適合測定器の処置と**遡及調査**（その器で測った製品の扱い）★
- [ ] 記録の保管

**校正計画表のテンプレ項目**
| 項目 | 記入内容の例 |
|---|---|
| 測定器ID・型式 | 管理番号、メーカー・型式 |
| 校正周期 | 〇か月／年（社内基準または製造元推奨） |
| 前回校正日・次回予定日 | 前回：〇年〇月〇日／次回：〇年〇月〇日 |
| 校正区分 | 社内校正／外部校正（校正機関名） |
| 基準器・トレーサビリティ | 使用した基準器の管理番号、国家標準への追跡可能性 |
| 判定 | 合格／要調整／不合格 |
| 使用中止・復帰の記録 | 中止日・理由、復帰日・確認者 |

**期限切れを発見したときの対応手順（テキストフロー）**
1. その場で測定器の使用を止める（校正切れのまま使い続けない）★
2. いつから期限切れだったかを特定する（前回校正日からの経過を確認）
3. その測定器で測定した製品・ロットを洗い出す（遡及調査の範囲を確定）★
4. 品証へ報告し、遡及調査の要否・範囲を判断してもらう
5. 校正を実施し、合格を確認してから使用を再開する
6. 校正切れが発生した原因（管理漏れ・台帳の不備等）を確認し、再発防止策を検討する

**NG例**
- [ ] 「前回問題なかったから」と校正切れのまま使い続ける★
- [ ] 遡及調査をせず、校正が済んだ時点から測定を再開する（切れていた期間の製品を確認しないまま流出させるリスク）★

関連場面: M4, T14, D1

## D4. 化学物質管理・法規対応
**目的**：適用法規と顧客要求を「聞かれる前に」満たし、法改正・材料変更があっても答えられる状態を保つ。

**ウォッチの型（担当・頻度を決めて回す）**
- REACH SVHC候補リストは年2回程度（おおむね1月・6月頃）更新される。更新の都度、自社の使用材料・購入品に該当がないか棚卸しする ★
- 国内の規制物質追加（化審法の第一種特定化学物質の指定、安衛法のSDS交付・リスクアセスメント対象物質の拡大など）は施行日が先に公表される。施行日から逆算して、SDS更新・該非判定・サプライヤーへの再調査依頼の期日を引く ★
- ウォッチ結果は「該当なし」も記録する。次に聞かれたときに、いつ・誰が・どの版で確認したかを即答できるのが目的
- 安衛法のリスクアセスメント対象物質は2026年4月から大幅に拡大し（従来の674物質から約2,900物質規模へ）、これまで対象外だった事業場でも化学物質管理者の選任が必要になっている。さらに2026年10月からは濃度基準値に基づく測定義務が始まるため、対象工程の洗い出しと測定体制の手配は前倒しで進める ★
- 社内での確認頻度・保存年限・回付先は組織ごとに異なるため 【要確認】

**よくあるつまずき**
- SDSを入手して満足し、含有判定・顧客回答まで結び付いていない
- 材料は同じでも副資材（離型剤・接着剤・インク・梱包材）が調査対象から漏れる ★
- 法改正時に「新規購入品」だけ見て、既存の継続購入品を見直していない

関連場面: T6, P4, V5, B4

## D5. 文書・記録管理
**目的**：「最新版はどれか」「その記録はどこにあるか」で迷わない状態を保ち、監査・顧客要求・不具合追跡のときに即座に根拠を出せるようにする。

- [ ] 文書の版管理（最新版のみ現場に・旧版回収）★
- [ ] 承認・配布・改廃のルール
- [ ] 記録の保管期間・保管場所・アクセス権限★
- [ ] トレーサビリティ（製品⇔ロット⇔記録）
- [ ] 機密・顧客情報・個人情報の取り扱い

**電子記録の管理（紙・電子が混在する時代の必須項目）**
- [ ] 正本を1つに決める（同じ記録が紙と電子の両方にあるとき、どちらが正かを明示する）★
- [ ] 編集履歴＝監査証跡（誰が・いつ・何を変えたか）が残る仕組みにする ★
- [ ] アクセス権限を「閲覧／編集／承認」で分け、退職・異動時に見直す
- [ ] バックアップと復旧手段を確認する（クラウド任せにせず、復元できるかを一度試す）
- [ ] 保存年限が過ぎた記録の廃棄ルールも決める（残し続けることが安全とは限らない）
- [ ] 電子化への移行時は、移行日・並行運用の終了日・過去記録の所在を文書に残す（TS27）★

**よくあるつまずき**
- 紙の様式をそのまま電子化しただけで、必須入力・入力チェックを組み込んでいない
- 共有フォルダに「最終版」「最終版2」が並び、どれが正本か分からなくなる

> 【要確認】記録様式・保管期間・アクセス権限・電子記録の要件（監査証跡・正本の定義）の社内規定。

関連場面: S2, S7, C4, TS27

## D6. KPI・品質目標の管理
**目的**：数字を「並べる」のではなく、数字で「打ち手を決める」。見ても何も変わらない指標は、指標ではない。

**設計（立ち上げ時）**
- [ ] 目標を設定（不良率・クレーム件数・是正リードタイム・コスト・納期・改善件数）
- [ ] 測定方法・定義を統一（分子・分母・集計期間・除外条件まで文章で固定する）★
- [ ] 指標は「行動につながる粒度」まで割る（総不良率ではなく、工程別・原因別・製品別に割れているか）★
- [ ] 各指標に「未達のときに誰が何をするか」を1行で書く。書けない指標は採用しない★

**運用（月次）**
- [ ] 月次でモニタし傾向分析（パレート・推移）。単月の上下ではなく傾向で判断する
- [ ] 未達時のアクションを決める（放置しない）★
- [ ] 「数字を見て変えた打ち手」を毎月1件以上、記録に残す（KPIが機能している唯一の証拠）★
- [ ] 全員で共有（自分ごと化）。現場ボード（A8）と定義・数字を一致させる
- [ ] 期ごとに目標を見直す（達成し続けている指標・誰も見ていない指標は入れ替える）

**よくあるつまずき**
- 定義が曖昧なまま運用し、部署ごとに違う数字が出て議論が止まる
- 指標が多すぎて全部が「見るだけ」になる（絞る勇気が要る）
- ダッシュボードを作った時点で満足し、打ち手が出ないまま形骸化する（TS28）

**KPIシート記入項目例（テンプレ）**
- 指標名／定義（分子・分母）／集計期間／目標値／実績値／差異／主要因／対応アクション／担当／期限

**記入例（良い例）**
不良率（工程A）：目標0.5%、実績0.9%。要因：新人作業者の検査見落とし2件。アクション：作業標準の再教育を今週中に実施、来月まで重点確認。担当：班長

**NG例**
不良率：目標未達。原因不明。来月も様子を見る。（→ 要因・アクション・担当・期限が無く、次に何をするか誰も分からない）

**未達時の判断フロー（テキスト）**
1. 傾向か単月かを確認する（3か月の推移を見る）。単月のブレなら継続監視にとどめる
2. 傾向として悪化しているなら、要因を工程別・原因別に分解する
3. 対応アクションと担当・期限を決め、翌月のKPIレビューで効果を確認する
4. 2か月連続未達かつ改善の効果が出ない場合は、指標の定義・目標値そのものを見直す

> 【要確認】KPIの正式定義・目標値・集計方法、未達時のアクションと責任者。

関連場面: A3, A8, D8, TS28

## D7. BCP・災害時の品質維持（緊急時の検査・記録）
**目的**：地震・停電・設備トラブル・感染症などの非常時でも、「出してはいけない物を出さない」最低限の品質を守る。

- [ ] 非常時に最優先で守る品質項目（安全・法規・重要特性）を事前に決める★
- [ ] 電源・測定器・システム停止時の代替手段（手記録・応急検査）を用意★
- [ ] 復旧後は、非常時ロットの追跡・再検査・隔離解除の判断基準を明確化
- [ ] 記録が取れなかった期間のトレーサビリティ確保（範囲の特定）
- [ ] 復旧手順・連絡網・代替供給を含むBCPを年1回は見直し・訓練
- [ ] 【要確認】非常時の必須品質項目・代替検査手段・再開判定の権限

関連場面: D5, B7, C1

## D8. 層別工程監査（LPA）・工程パトロールの運用
**目的**：問題が起きる前に現場のズレを見つける。役職を横断した“層別”の定期監査で、標準どおりの作業・記録・現物を確認する。

- [ ] 重要工程を対象に、確認項目を絞ったチェックシートを用意する★
- [ ] リーダー・管理者など層を分けて定期的に巡回・確認する（層別監査）★
- [ ] 標準・記録・現物・5S・測定器の状態を“その場で”確認する
- [ ] 逸脱はその場で是正し、繰り返すものは仕組み（標準・ポカヨケ）を直す
- [ ] 指摘・是正の記録を残し、傾向を月次でレビューする（D6と連動）

関連場面: D3, D6, M8, M6

## D9. 外部試験機関の活用（依頼試験・成績書・技能試験）
**目的／考え方**：自社で実施できない試験（特殊な物性試験・法規適合試験等）は外部試験機関に依頼する。依頼先の選定・成績書の信頼性確認・技能試験（外部評価）への参加を通じて、測定の妥当性を担保する。

- [ ] 依頼する試験項目・規格・判定基準を明確にしてから依頼する★
- [ ] 依頼先の認定資格（ISO/IEC 17025等）や実績を確認し、妥当性のある機関を選定する★
- [ ] 依頼時にサンプルの状態（ロット・保管条件・輸送方法）が結果に影響しないよう管理する
- [ ] 成績書の記載内容（規格・実測値・不確かさ・判定）を受領時に確認する★
- [ ] 定期的に技能試験（外部の評価プログラム）に参加し、自社測定の妥当性を確認する
- [ ] 技能試験の結果が思わしくない場合は、原因究明・是正を行い記録に残す★
- [ ] 依頼・結果の記録を保管し、監査・顧客提出時に参照できるようにする
- [ ] 【要確認】主要な依頼先試験機関のリスト、技能試験の参加頻度・対象項目

関連場面: T14, D2, D3

## D10. 品質コスト（COQ）の見える化と改善テーマ選定
**目的／考え方**：品質活動は感覚だけで評価されがちだが、予防・評価・失敗の各コストを見える化すると、どこに手を打つべきかが数字で見えてくる。COQ（Cost of Quality）を集計し、改善テーマの優先順位づけに使う。

- [ ] コストを3区分（予防コスト・評価コスト・失敗コスト＝内部／外部）に分けて集計する★
- [ ] 失敗コスト（不良・クレーム・返品・手直し工数等）を優先して可視化する（インパクトが大きいため）★
- [ ] 集計は既存のKPI（D6）・不良記録・クレーム記録のデータを流用し、新たな入力業務を極力増やさない
- [ ] 部門・工程別にCOQを比較し、改善テーマの候補を絞り込む★
- [ ] 改善テーマは「削減額の見込み」と「対応工数」で優先順位をつける
- [ ] 改善実施後は、失敗コストの減少で効果を確認する（B10有効性確認と連携）
- [ ] 経営層への報告は、金額換算した数字で説明する（品質を「コスト」の言葉で伝える）
- [ ] 【要確認】コスト区分の集計方法・単価の考え方、経営報告の頻度・様式

関連場面: D6, B10, A13

# E. 教育・OJT

> 位置づけ：**A5「教育・力量管理／新任受け入れ」**は管理者（GL）が力量管理の仕組みを運営する話（スキルマップの設計・新任受け入れ手順・外部研修の判断）。**E章**は「教える側・学ぶ側」がその場でどう動くかという実務そのもの（OJTの計画の立て方・教え方の型・力量の見える化・資格管理・技能の残し方）を扱う。A5＝仕組み、E＝現場実務、とセットで運用する。

## E1. OJT計画の立て方（対象者・期間・到達基準・チェックリスト）
**目的**：思いつきで教えず、「誰に・何を・いつまでに・どこまでできれば良いか」を最初に決めてから教え始める。

- [ ] 対象者と現在の力量（何ができて、何ができないか）を確認する ★
- [ ] 到達基準を具体的な行動で書く（「わかる」ではなく「一人で〇〇ができる」）★
- [ ] 期間を区切る（短期スキルは週単位、資格系は月単位など目安を決める）
- [ ] 教える内容を小さい単位に分割し、順番を決める（安全・基本→応用の順）
- [ ] 教える人（指導担当）を1人決め、進捗の確認者（GL等）を分ける
- [ ] 中間チェックのタイミングを計画に入れる（詰め込みで最後にまとめて確認しない）★
- [ ] 独り立ち判定の基準と判定者を計画時点で決めておく（A5と連動）★

**OJT計画表に入れる項目（テンプレ）**
| 項目 | 記入内容の例 |
|---|---|
| 対象者・開始日 | 氏名、配属日、経験の有無 |
| 到達基準 | 「〇〇工程の検査を一人で実施し、記録まで正しく残せる」等、行動で書く |
| 教える内容と順番 | 安全→基本ルール→単位作業→応用・異常時対応 |
| 期間・中間チェック日 | 全体〇週間、中間チェックを週1で実施 |
| 指導担当・確認者 | 指導：〇〇／進捗確認：GL |
| 独り立ち判定基準 | 誰が・何を見て・どの基準で合格とするか |

> 【要確認】OJT計画表の正式様式・保存先、独り立ち判定の正式な権限者。

関連場面: A5, E2

## E2. 仕事の教え方の基本（やってみせる→説明する→やらせてみる→フォロー。つまずき別の対処）
**目的**：勘や根性ではなく、型で教える。同じ教え方をすれば、誰が教えても質が揃う。

**基本の4ステップ**
1. **やってみせる**：まず指導者が実際にやって見せる（説明を先にしすぎない）★
2. **説明する**：急所（間違えやすい点・安全上のポイント）を理由つきで説明する ★
3. **やらせてみる**：本人にやらせ、その場で間違いを直す（黙って見ているだけにしない）
4. **フォローする**：一人でやらせたあと、質問できる状態にしておく（放置しない）★

**つまずき別の対処**
- [ ] 手順は覚えているが遅い → 反復の回数が不足。時間を測って本人にも見せる
- [ ] 急所（コツ）を守れない → 「なぜそれが急所か」の理由から教え直す（手順の丸暗記だけでは崩れる）★
- [ ] 教えた通りにやらない（自己流になる） → その場で止めて理由を確認し、正しい理由を再説明する
- [ ] 質問してこない → 「聞いても良い」空気を最初に伝え、定期的にこちらから聞きに行く ★
- [ ] 覚えたはずが数週間後にできない → フォロー頻度が足りない。中間チェックの回数を増やす

**やってはいけない（NG）**
- 「見て覚えて」で説明を省略する ★
- 一度で全部を教え、質問の時間を取らない

関連場面: A5, E1

## E3. 力量マップの作り方・運用（評価基準・更新タイミング・見える化）
**目的**：誰が何をどのレベルでできるかを一覧化し、教育計画・多能工化・応援配置の判断にそのまま使える状態にする。感覚ではなく、見える化された基準で運用する。

- [ ] 対象工程・作業を洗い出し、必要なスキル項目に分解する（ひとまとめにしすぎない）★
- [ ] レベルの定義を段階で決める（例：4段階＝未経験／指導があればできる／一人でできる／人に教えられる）★
- [ ] 評価基準を行動で書く（「知っている」ではなく「一人で〇〇を規定通りにできる」）
- [ ] 評価者と評価タイミングを決める（自己評価だけで終わらせず、指導担当・GLが確認する）
- [ ] 更新タイミングを決める（配置転換・資格取得・一定期間経過時など）★
- [ ] 見える化の方法を決める（掲示・共有ファイル等、本人と周囲が確認できる状態にする）
- [ ] 力量マップを教育計画（E1）・多能工化・応援配置の判断に使う運用ルールを明記する ★

**レベル定義の例**
| レベル | 状態の目安 |
|---|---|
| 1 | 未経験・指導を受けている段階 |
| 2 | 指導者がそばにいればできる |
| 3 | 一人で規定通りにできる |
| 4 | 人に教えられる（指導担当になれる） |

> 【要確認】評価基準・レベル定義の正式な文言、更新頻度の正式ルール、掲示・共有の方法。

関連場面: A5, E1, M9

## E4. 新任者の30-60-90日プラン（品証配属者向けの到達目標）
**目的**：新任者が「何をいつまでにできればよいか」を段階で示し、教える側も進捗を同じものさしで確認できるようにする。

- [ ] 30日目の到達目標を決める（基本ルール・安全・報告連絡相談の型を理解し、決められた作業を指導者と一緒にできる）★
- [ ] 60日目の到達目標を決める（頻度の高い基本作業を一人でできる。異常時は必ず報告できる）★
- [ ] 90日目の到達目標を決める（担当領域の主要業務を一人で回せる。簡単な判断は自分で下せる）
- [ ] 各節目で振り返りの場を設ける（本人の自己評価と、指導担当・GLの確認をセットで行う）★
- [ ] 到達できていない項目があれば、原因を確認しプランを調整する（期間延長・再指導）
- [ ] 90日終了時点の力量を力量マップ（E3）に反映する
- [ ] プランは配属前のオリエンテーションで本人に共有し、ゴールを最初から明示する ★

**到達目標の目安**
| 期間 | 到達目標の例 |
|---|---|
| 〜30日 | 基本ルール・安全・報連相の型を理解し、指導者と一緒に基本作業ができる |
| 〜60日 | 頻度の高い基本作業を一人でできる。異常時は必ず報告できる |
| 〜90日 | 担当領域の主要業務を一人で回せる。簡単な判断は自分で下せる |

> 【要確認】配属先別の到達目標の詳細、正式なオリエンテーション資料の有無。

関連場面: A5, E1, E3

## E5. 認定検査員・資格の管理（認定基準・更新・一時停止の判断）
**目的**：検査員の力量を「認定」という形で管理し、誰がどの検査を実施してよいかを明確にする。認定切れ・力量不足のまま検査を継続させない。

- [ ] 検査項目ごとに認定が必要な範囲を決める（全作業ではなく、判定が難しい・重要度の高い検査から優先）★
- [ ] 認定基準を決める（実技確認・知識確認・合否判定者を明記）★
- [ ] 認定の有効期間と更新方法を決める（更新前研修・再評価の要否）
- [ ] 認定者一覧を作り、本人・GL・品証など誰が確認しても分かる状態にする
- [ ] 一時停止の基準を決める（誤判定・長期休務明け・工程変更後など、無条件で継続させない場合を明記）★
- [ ] 一時停止からの復帰基準を決める（再評価に合格するまで対象検査を実施させない）
- [ ] 外注先の検査員についても同じ考え方を適用する（力量確認・教育の依頼と連携）★

**状態別の対応**
| 状態 | 対応 |
|---|---|
| 認定中 | 対象検査を実施可 |
| 更新期限接近 | 更新研修・再評価を計画 |
| 一時停止 | 対象検査から外し、再評価まで実施させない |
| 再評価合格 | 認定を再開し、一覧を更新 |

> 【要確認】認定基準・有効期間の正式な数値、一時停止の権限者。外注先検査員の認定管理を自社基準のまま適用するか、取引先ごとに確認するかは別途判断。

関連場面: A5, E1, E3, V11

## E6. 技能伝承・マニュアル化（動画・写真基準書の作り方と更新ルール）
**目的**：熟練者の勘・コツを個人任せにせず、動画・写真基準書という形に残し、次の世代が同じ基準で再現できるようにする。

- [ ] 対象作業を選ぶ（習得に時間がかかる・属人化している・引き継ぎリスクが高い作業を優先）★
- [ ] 撮影・作成前に急所（コツ・NG動作）を熟練者から聞き取り、言語化しておく ★
- [ ] 動画は「全体の流れ」と「急所の拡大」を分けて撮る（急所だけ早口で終わらせない）
- [ ] 写真基準書は良品・NG品の対比が分かる構成にする（数値基準がある場合は基準値を明記）
- [ ] 作成物は指導担当・GLがレビューし、事実と基準に相違がないか確認する ★
- [ ] 保管場所・閲覧方法を決め、教育（E1・E2）で実際に使う運用にする
- [ ] 更新タイミングを決める（工程変更・設備更新・基準改訂のたびに見直す）★

> 【要確認】撮影・保管の正式な運用ルール（保存先・命名規則・アクセス権限）、更新の責任者。

関連場面: A5, E1, E2

## E7. 中途採用者・応援者への短期教育（最低限教える項目・確認方法。M9多能工化と連携）
**目的**：期間が限られる中途採用者・応援者に、フル教育ではなく「最低限、安全に品質を守って作業できる」範囲を短時間で確実に教える。

- [ ] 対象者の経験・前歴を確認し、教える範囲を絞る（未経験前提の全教育をしない）★
- [ ] 最低限教える項目を決める（安全・止める基準・異常時の報告先・担当作業の急所）★
- [ ] 教える順番を決める（安全→止める基準→担当作業の基本→異常時対応）
- [ ] 短期間でも到達確認を行う（教えっぱなしにせず、本人にやらせて確認する）★
- [ ] 応援期間中の力量は力量マップ（E3）に暫定記録し、期間終了時に扱いを決める
- [ ] 独り立ちさせてよい範囲と、必ず指導者が付く範囲を分けて明示する
- [ ] 多能工化の考え方と揃え、応援先の工程でも同じ基準で確認する ★

> 【要確認】応援者・中途採用者向けの標準教育資料の有無、期間終了時の力量記録の扱い。

関連場面: A5, E1, E3, M9
