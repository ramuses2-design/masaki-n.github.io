# 品質保証グループ ハンドブック — 運用手順

## 正（Single Source of Truth）
本ハンドブックの本文は、以下2つの Markdown が**唯一の正**です。サイトはそれを反映したものです。

- `content/policy-master.md` … 品質保証グループ 方針マスター（使命・価値観・ルール・エスカレーション基準・用語）
- `content/playbooks.md` … 場面別プレイブック集（A〜D / 全23場面の手順・チェックリスト）

## 内容を変えるとき（必ずこの流れ）
1. **Markdown を直す**（上記2ファイルのどちらか。サイトのHTMLは直接編集しない）
2. **再生成**：`python3 tools/gen_handbook.py`（下記）を実行し、`/handbook/` 配下のHTML・検索インデックスを作り直す
3. **PR / push** で公開する

> HTML を手で書き換えると、次回の再生成で上書きされます。必ず Markdown 側を直してください。

## 凡例
- `★` … 特に外すと事故につながる**重点項目**。サイトでは塗りつぶしチェックボックス＋「重点」バッジで強調表示されます。
- `【要確認】` … GL（masaki）の確定が必要な**未確定**箇所。サイトでは淡色の「要確認」バッジで表示し、`要確認リスト` ページに集約しています。確定したら Markdown 側のタグを外してください。

## 生成スクリプト
`tools/gen_handbook.py`（このリポジトリの `handbook/tools/` に同梱）が、2つの Markdown を読み込み、
ハブ／グループ（A〜D）／各場面ページ／方針／用語・索引／要確認リスト／全文検索インデックスを生成します。
外部ライブラリ不要（Python標準のみ）。検索はクライアントサイドの軽量JS（`assets/app.js` + `assets/search-index.json`）です。

## 触らないもの
- ルートの `index.html`・`data.json`・`archive/`（= Daily Insight Board）。本ハンドブックは `/handbook/` 配下に独立しており、ボードの自動更新には影響しません。
