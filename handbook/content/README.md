# 品質保証グループ ハンドブック — 運用手順

## 正（Single Source of Truth）
本ハンドブックの本文は、以下4つの Markdown が**唯一の正**です。サイトはそれを反映したものです。

- `content/policy-master.md` … 方針マスター（使命・価値観・ルール・エスカレーション基準・用語）
- `content/playbooks.md` … 場面別プレイブック集（A〜D / 全23場面）
- `content/audience-guides.md` … 読者別ガイド（S 営業 / T 技術 / M 製造 / P 購買 / V 外注先）
- `content/troubleshooting.md` … トラブルシューティング（症状ベースの不良対応・たたき台。TS◯）

## 内容を変えるとき（必ずこの流れ）
1. **Markdown を直す**（上記4ファイルのいずれか。サイトのHTMLは直接編集しない）
2. **再生成**：`python3 tools/gen_handbook.py` を実行し、`/handbook/` 配下のHTML・検索インデックスを作り直す
3. **PR / push** で公開する

> HTML を手で書き換えると、次回の再生成で上書きされます。必ず Markdown 側を直してください。

## 表示（サイドバー型）
- 左サイドバーに全項目のツリー（方針／場面別A〜D／読者別／トラブルシューティング／リファレンス）を常時表示。
- 各セクションは折りたたみ（`<details>`）で、現在地の節が自動で開き、該当ページがハイライトされます。
- スマホでは左上の「☰」でサイドバーを開閉。上部の検索は全ページ共通。

## 凡例
- `★` … 特に外すと事故につながる**重点項目**。塗りつぶしチェック＋「重点」バッジ。
- `【要確認】` … GL（masaki）の確定が必要な**未確定**箇所。`要確認リスト` ページに集約。確定したら Markdown 側のタグを外す。
- 読者別・トラブルの各項目末尾 `関連場面: B2, TS1 …` … 場面（A〜D）や症状（TS）への相互リンクになる（IDで指定）。

## 追加のしかた（例）
- 読者別トピックを増やす：`audience-guides.md` の該当読者セクションに `## S9. タイトル` を追記 → 再生成。
- 症状を増やす：`troubleshooting.md` に `## TS13. タイトル` を追記 → 再生成。
- 新しい読者を増やす：`gen_handbook.py` の `AUD` に1行追加し、`audience-guides.md` に `# X. 〇〇向け` を追記。

## 触らないもの
- ルートの `index.html`・`data.json`・`archive/`（= Daily Insight Board）。本ハンドブックは `/handbook/` 配下に独立しており、ボードの自動更新には影響しません。
